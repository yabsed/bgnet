4 IPv4에서 IPv6로 넘어가기

하지만 제 코드를 IPv6에서 돌아가게 하려면 무엇을 바꿔야 하는지만 알고 싶어요! 지금 바로 알려주세요!

알겠어요! 알겠어요!

여기에 있는 거의 모든 내용은 위에서 이미 설명한 것이지만, 조급한 사람들을 위한 짧은 버전입니다. (물론 이보다 더 많은 내용이 있지만, 이 가이드에 해당하는 건 이 정도입니다.)

  1. 무엇보다도, 구조체를 손으로 직접 채워 넣는 대신 getaddrinfo()를 사용해 모든 struct sockaddr 정보를 얻도록 해보세요. 이렇게 하면 IP 버전에 독립적인 코드를 유지할 수 있고, 뒤따르는 여러 단계를 생략할 수 있습니다.

  2. IP 버전과 관련된 값을 하드코딩한 곳을 발견하면, 가능하면 헬퍼 함수로 감싸 두세요.

  3. AF_INET을 AF_INET6으로 바꾸세요.

  4. PF_INET을 PF_INET6으로 바꾸세요.

  5. INADDR_ANY 대입은 약간 방식이 다른 in6addr_any 대입으로 바꾸세요:

    struct sockaddr_in sa;
struct sockaddr_in6 sa6;

sa.sin_addr.s_addr = INADDR_ANY;  // 내 IPv4 주소 사용
sa6.sin6_addr = in6addr_any; // 내 IPv6 주소 사용

    또한 struct in6_addr를 선언할 때는 IN6ADDR_ANY_INIT 값을 다음과 같이 초기화 값으로 사용할 수 있습니다:

    struct in6_addr ia6 = IN6ADDR_ANY_INIT;

  6. struct sockaddr_in 대신 struct sockaddr_in6을 사용하고, 필드 이름에도 알맞게 “6”을 붙이세요(위의 struct 설명 참고). sin6_zero 필드는 없습니다.

  7. struct in_addr 대신 struct in6_addr를 사용하고, 필드 이름에도 알맞게 “6”을 붙이세요(위의 struct 설명 참고).

  8. inet_aton() 또는 inet_addr() 대신 inet_pton()을 사용하세요.

  9. inet_ntoa() 대신 inet_ntop()을 사용하세요.

  10. gethostbyname() 대신 더 나은 getaddrinfo()를 사용하세요.

  11. gethostbyaddr() 대신 더 나은 getnameinfo()를 사용하세요(gethostbyaddr()도 IPv6에서 여전히 동작할 수는 있습니다).

  12. INADDR_BROADCAST는 더 이상 작동하지 않습니다. 대신 IPv6 멀티캐스트를 사용하세요.

이상입니다!
