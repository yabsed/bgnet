9 man 페이지

유닉스 세계에는 매뉴얼이 아주 많습니다. 그 안에는 여러분이 사용할 수 있는 개별 함수들을 설명하는 작은 절들이 들어 있습니다.

물론 manual이라고 쓰기에는 너무 길죠. 제 말은, 저를 포함해서 유닉스 세계 사람들 중 그렇게 많이 타이핑하길 좋아하는 사람은 없다는 겁니다. 사실 제가 얼마나 짧게 쓰는 걸 좋아하는지 끝도 없이 길게 떠들 수도 있겠지만, 제가 모든 상황에서 얼마나 놀라울 정도로 간결함을 선호하는지에 대한 장황한 연설로 여러분을 지루하게 만들지는 않겠습니다.

[박수]

감사합니다. 제가 하려는 말은, 유닉스 세계에서는 이런 페이지를 “man 페이지”라고 부른다는 것입니다. 그리고 여러분이 읽기 편하도록 제 개인용 축약판을 여기 실어 두었습니다. 다만 이런 함수들 중 상당수는 제가 여기서 보여 주는 것보다 훨씬 더 범용적이고, 저는 그중에서도 인터넷 소켓 프로그래밍과 관련 있는 부분만 소개하려고 합니다.

하지만 잠깐! 제 man 페이지에는 이것 말고도 문제가 더 있습니다.

  • 불완전하며, 이 가이드에 나온 기본 내용만 보여 줍니다.
  • 실제 세상에는 이것보다 훨씬 더 많은 man 페이지가 있습니다.
  • 여기 내용은 여러분 시스템의 man 페이지와 다를 수 있습니다.
  • 어떤 함수는 여러분 시스템에서 헤더 파일이 다를 수 있습니다.
  • 어떤 함수는 여러분 시스템에서 파라미터가 다를 수 있습니다.

진짜 정보가 필요하다면 로컬 유닉스 man 페이지를 확인하세요. man whatever라고 입력하면 되는데, 여기서 “whatever”는 여러분이 몹시 관심 있는 무언가입니다. 예를 들면 “accept” 같은 것이죠. (Microsoft Visual Studio에도 도움말 섹션에 비슷한 게 분명 있겠죠. 하지만 “man”이 “help”보다 1바이트 더 짧으니 더 낫습니다. 또다시 유닉스의 승리입니다!)

그렇다면 이렇게 허점이 많은데 왜 굳이 가이드에 포함했을까요? 이유는 몇 가지 있지만, 가장 큰 이유는 (a) 이 버전들은 네트워크 프로그래밍에 맞춰져 있어서 실제 man 페이지보다 소화하기 쉽고, (b) 예제가 들어 있기 때문입니다!

아, 그리고 예제 이야기가 나와서 말인데, 저는 코드 길이가 너무 늘어나는 걸 막으려고 에러 검사를 전부 넣지는 않는 편입니다. 하지만 어떤 시스템 호출이든 실패하지 않는다고 100% 확신하는 경우가 아니라면 거의 항상 에러 검사를 해야 하고, 사실 그런 경우라도 하는 편이 좋습니다!

9.1 accept()

리스닝 소켓에서 들어오는 연결을 수락한다

형식

#include <sys/types.h>
#include <sys/socket.h>

int accept(int s, struct sockaddr *addr, socklen_t *addrlen);

설명

SOCK_STREAM 소켓을 만들고 listen()으로 들어오는 연결을 받을 준비까지 마쳤다면, 이제 accept()를 호출해서 새로 연결된 클라이언트와 이후 통신에 사용할 새 소켓 디스크립터를 실제로 얻습니다.

리스닝에 사용하던 기존 소켓은 그대로 남아 있고, 앞으로 들어오는 추가 연결에 대해 계속 accept()를 호출하는 데 사용됩니다.

파라미터 설명
s listen() 중인 소켓 디스크립터입니다.
addr 여기에는 여러분에게 연결을 시도한 상대의 주소가 채워집니다.
addrlen 여기에는 addr 파라미터로 돌려받은 구조체의 sizeof() 값이 채워집니다. addr로 넘긴 타입이 struct sockaddr_in이고 실제로도 그것이 돌아올 것이라고 알고 있다면, 이 값은 무시해도 안전합니다.

accept()는 보통 블록됩니다. 미리 리스닝 소켓 디스크립터를 살펴 “읽을 준비가 되었는지” 확인하려면 select()를 사용할 수 있습니다. 준비가 되어 있다면, accept()되기를 기다리는 새 연결이 있다는 뜻입니다! 야호! 또는 fcntl()로 리스닝 소켓에 O_NONBLOCK 플래그를 설정할 수도 있습니다. 그러면 블록되지 않고, 대신 errno를 EWOULDBLOCK으로 설정한 채 -1을 반환합니다.

accept()가 반환하는 소켓 디스크립터는 진짜배기 소켓 디스크립터로, 열린 상태이며 원격 호스트와 연결되어 있습니다. 사용이 끝나면 반드시 close()해야 합니다.

반환값

accept()는 새로 연결된 소켓 디스크립터를 반환합니다. 오류가 발생하면 -1을 반환하고, errno가 알맞게 설정됩니다.

예제

struct sockaddr_storage their_addr;
socklen_t addr_size;
struct addrinfo hints, *res;
int sockfd, new_fd;

// first, load up address structs with getaddrinfo():

memset(&hints, 0, sizeof hints);
hints.ai_family = AF_UNSPEC;  // use IPv4 or IPv6, whichever
hints.ai_socktype = SOCK_STREAM;
hints.ai_flags = AI_PASSIVE;     // fill in my IP for me

getaddrinfo(NULL, MYPORT, &hints, &res);

// make a socket, bind it, and listen on it:

sockfd = socket(res->ai_family, res->ai_socktype, res->ai_protocol);
bind(sockfd, res->ai_addr, res->ai_addrlen);
listen(sockfd, BACKLOG);

// now accept an incoming connection:

addr_size = sizeof their_addr;
new_fd = accept(sockfd, (struct sockaddr *)&their_addr, &addr_size);

// ready to communicate on socket descriptor new_fd!

9.2 bind()

소켓을 IP 주소와 포트 번호에 연결한다

형식

#include <sys/types.h>
#include <sys/socket.h>

int bind(int sockfd, struct sockaddr *my_addr, socklen_t addrlen);

설명

원격 머신이 여러분의 서버 프로그램에 연결하려면 두 가지 정보가 필요합니다. IP 주소와 포트 번호입니다. bind() 호출은 바로 그 일을 하게 해 줍니다.

먼저 getaddrinfo()를 호출해서 목적지 주소와 포트 정보가 들어 있는 struct sockaddr를 준비합니다. 그런 다음 socket()으로 소켓 디스크립터를 얻고, 그 소켓과 주소를 bind()에 넘기면 IP 주소와 포트가 마법처럼(진짜 마법을 써서) 소켓에 묶입니다!

자신의 IP 주소를 모르거나, 머신에 IP 주소가 하나뿐이라는 걸 알거나, 그 머신의 어떤 IP 주소를 쓸지 신경 쓰지 않는다면, getaddrinfo()의 hints 파라미터에 AI_PASSIVE 플래그를 넘기기만 하면 됩니다. 그러면 struct sockaddr의 IP 주소 부분에 특별한 값이 채워지고, bind()는 이를 보고 이 호스트의 IP 주소를 자동으로 채워 넣습니다.

뭐라고요? 현재 호스트 주소가 자동으로 채워지게 만드는 특별한 값이 struct sockaddr의 IP 주소에 정확히 무엇으로 들어가느냐고요? 알려 드리죠. 다만 이건 struct sockaddr를 손으로 채울 때 이야기입니다. 그렇지 않다면 위에서처럼 getaddrinfo() 결과를 쓰면 됩니다. IPv4에서는 struct sockaddr_in 구조체의 sin_addr.s_addr 필드를 INADDR_ANY로 설정합니다. IPv6에서는 struct sockaddr_in6 구조체의 sin6_addr 필드에 전역 변수 in6addr_any를 대입합니다. 또는 새 struct in6_addr를 선언하는 경우 IN6ADDR_ANY_INIT으로 초기화할 수 있습니다.

마지막으로 addrlen 파라미터는 sizeof my_addr로 설정해야 합니다.

반환값

성공하면 0을, 오류가 발생하면 -1을 반환합니다. 이때 errno가 알맞게 설정됩니다.

예제

// modern way of doing things with getaddrinfo()

struct addrinfo hints, *res;
int sockfd;

// first, load up address structs with getaddrinfo():

memset(&hints, 0, sizeof hints);
hints.ai_family = AF_UNSPEC;  // use IPv4 or IPv6, whichever
hints.ai_socktype = SOCK_STREAM;
hints.ai_flags = AI_PASSIVE;     // fill in my IP for me

getaddrinfo(NULL, "3490", &hints, &res);

// make a socket:
// (you should actually walk the "res" linked list and error-check!)

sockfd = socket(res->ai_family, res->ai_socktype, res->ai_protocol);

// bind it to the port we passed in to getaddrinfo():

bind(sockfd, res->ai_addr, res->ai_addrlen);
// example of packing a struct by hand, IPv4

struct sockaddr_in myaddr;
int s;

myaddr.sin_family = AF_INET;
myaddr.sin_port = htons(3490);

// you can specify an IP address:
inet_pton(AF_INET, "63.161.169.137", &(myaddr.sin_addr));

// or you can let it automatically select one:
myaddr.sin_addr.s_addr = INADDR_ANY;

s = socket(PF_INET, SOCK_STREAM, 0);
bind(s, (struct sockaddr*)&myaddr, sizeof myaddr);

9.3 connect()

소켓을 서버에 연결한다

형식

#include <sys/types.h>
#include <sys/socket.h>

int connect(int sockfd, const struct sockaddr *serv_addr,
            socklen_t addrlen);

설명

socket() 호출로 소켓 디스크립터를 만들었다면, 이름 그대로인 connect() 시스템 호출을 사용해 그 소켓을 원격 서버에 연결할 수 있습니다. 해야 할 일은 소켓 디스크립터와, 좀 더 가까이 알고 싶은 서버의 주소를 넘기는 것뿐입니다. (아, 그리고 이런 함수들에 흔히 함께 넘기는 주소 길이도요.)

보통 이 정보는 getaddrinfo() 호출 결과로 얻지만, 원한다면 직접 struct sockaddr를 채워 넣어도 됩니다.

아직 그 소켓 디스크립터에 대해 bind()를 호출하지 않았다면, 여러분의 IP 주소와 임의의 로컬 포트에 자동으로 바인드됩니다. 서버가 아니라면 대개 이걸로 충분합니다. 로컬 포트가 무엇인지는 별로 중요하지 않고, serv_addr 파라미터에 넣어야 하는 원격 포트만 중요하기 때문입니다. 정말로 클라이언트 소켓을 특정 IP 주소와 포트에 두고 싶다면 bind()를 호출할 수는 있지만, 그런 경우는 꽤 드뭅니다.

소켓이 connect()되면 마음껏 send()와 recv()로 데이터를 주고받을 수 있습니다.

특별 참고: SOCK_DGRAM UDP 소켓을 원격 호스트에 connect()하면, 원한다면 sendto()와 recvfrom()뿐 아니라 send()와 recv()도 사용할 수 있습니다.

반환값

성공하면 0을, 오류가 발생하면 -1을 반환합니다. 이때 errno가 알맞게 설정됩니다.

예제

// connect to www.example.com port 80 (http)

struct addrinfo hints, *res;
int sockfd;

// first, load up address structs with getaddrinfo():

memset(&hints, 0, sizeof hints);
hints.ai_family = AF_UNSPEC;  // use IPv4 or IPv6, whichever
hints.ai_socktype = SOCK_STREAM;

// we could put "80" instead on "http" on the next line:
getaddrinfo("www.example.com", "http", &hints, &res);

// make a socket:

sockfd = socket(res->ai_family, res->ai_socktype, res->ai_protocol);

// connect it to the address and port we passed in to getaddrinfo():

connect(sockfd, res->ai_addr, res->ai_addrlen);

9.4 close()

소켓 디스크립터를 닫는다

형식

#include <unistd.h>

int close(int s);

설명

여러분이 꾸며 낸 그 기상천외한 계획을 위해 소켓을 다 사용했고, 더 이상 그 소켓으로 send()도 recv()도, 아니 사실상 아무것도 하고 싶지 않다면, close()를 호출하면 됩니다. 그러면 소켓은 해제되고 다시는 사용되지 않습니다.

원격 측은 이 상황을 두 가지 방식으로 알 수 있습니다. 첫째, 원격 측이 recv()를 호출하면 0을 반환합니다. 둘째, 원격 측이 send()를 호출하면 SIGPIPE 시그널을 받고, send()는 -1을 반환하며 errno는 EPIPE로 설정됩니다.

Windows 사용자: 이때 써야 하는 함수는 close()가 아니라 closesocket()입니다. 소켓 디스크립터에 close()를 쓰려고 하면 Windows가 화를 낼 수도 있습니다. 그리고 Windows가 화났을 때는 별로 보고 싶지 않을 겁니다.

반환값

성공하면 0을, 오류가 발생하면 -1을 반환합니다. 이때 errno가 알맞게 설정됩니다.

예제

s = socket(PF_INET, SOCK_DGRAM, 0);
.
.
.
// a whole lotta stuff...*BRRRONNNN!*
.
.
.
close(s);  // not much to it, really.

9.5 getaddrinfo(), freeaddrinfo(), gai_strerror()

호스트 이름 및/또는 서비스 정보를 가져와 결과로 struct sockaddr를 채운다.

형식

#include <sys/types.h>
#include <sys/socket.h>
#include <netdb.h>

int getaddrinfo(const char *nodename, const char *servname,
                const struct addrinfo *hints,
                struct addrinfo **res);

void freeaddrinfo(struct addrinfo *ai);

const char *gai_strerror(int ecode);

struct addrinfo {
  int     ai_flags;          // AI_PASSIVE, AI_CANONNAME, ...
  int     ai_family;         // AF_xxx
  int     ai_socktype;       // SOCK_xxx
  int     ai_protocol;       // 0 (auto) or IPPROTO_TCP, IPPROTO_UDP 

  socklen_t  ai_addrlen;     // length of ai_addr
  char   *ai_canonname;      // canonical name for nodename
  struct sockaddr  *ai_addr; // binary address
  struct addrinfo  *ai_next; // next structure in linked list
};

설명

getaddrinfo()는 특정 호스트 이름에 대한 정보(예: IP 주소)를 반환하고, IPv4인지 IPv6인지 같은 자잘한 세부사항까지 처리해 주면서 struct sockaddr를 채워 주는 훌륭한 함수입니다. 예전의 gethostbyname()과 getservbyname()를 대체합니다. 아래 설명에는 조금 벅찰 수 있는 정보가 많이 들어 있지만, 실제 사용법은 꽤 단순합니다. 먼저 예제를 보는 것도 좋습니다.

관심 있는 호스트 이름은 nodename 파라미터에 넣습니다. 주소는 “www.example.com” 같은 호스트 이름일 수도 있고, 문자열로 전달하는 IPv4 또는 IPv6 주소일 수도 있습니다. AI_PASSIVE 플래그를 사용할 경우(아래 참고) 이 파라미터는 NULL이어도 됩니다.

servname 파라미터는 기본적으로 포트 번호입니다. “80”처럼 문자열 형태의 포트 번호일 수도 있고, “http”, “tftp”, “smtp”, “pop” 같은 서비스 이름일 수도 있습니다. 잘 알려진 서비스 이름은 IANA Port List49나 /etc/services 파일에서 찾을 수 있습니다.

마지막 입력 파라미터는 hints입니다. 이 구조체에서 getaddrinfo()가 정확히 무엇을 하게 할지 정의합니다. 사용하기 전에 memset()으로 구조체 전체를 0으로 초기화하세요. 이제 사용 전에 설정해야 하는 필드를 보겠습니다.

ai_flags에는 여러 값을 넣을 수 있는데, 여기서는 중요한 것 몇 가지만 보겠습니다. 플래그가 여러 개라면 | 연산자로 비트 OR 해서 함께 지정할 수 있습니다. 전체 목록은 로컬 man 페이지를 확인하세요.

AI_CANONNAME은 결과의 ai_canonname에 호스트의 canonical(실제) 이름을 채우게 합니다. AI_PASSIVE는 결과의 IP 주소를 INADDR_ANY(IPv4) 또는 in6addr_any(IPv6)로 채우게 합니다. 그러면 뒤이어 호출되는 bind()가 struct sockaddr의 IP 주소를 현재 호스트 주소로 자동 채워 넣게 됩니다. 주소를 하드코딩하고 싶지 않을 때 서버를 준비하기에 아주 좋습니다.

AI_PASSIVE 플래그를 사용하는 경우에는 nodename에 NULL을 넘겨도 됩니다. 나중에 bind()가 대신 채워 넣기 때문입니다.

입력 파라미터를 계속 보면, 보통 ai_family를 AF_UNSPEC으로 설정하게 됩니다. 그러면 getaddrinfo()가 IPv4와 IPv6 주소를 모두 찾습니다. AF_INET이나 AF_INET6로 설정해 한쪽으로 제한할 수도 있습니다.

다음으로 socktype 필드는 원하는 소켓 타입에 따라 SOCK_STREAM 또는 SOCK_DGRAM으로 설정해야 합니다.

마지막으로 ai_protocol은 0으로 두면 프로토콜 종류가 자동으로 선택됩니다.

이제 필요한 값을 다 채웠다면, 드디어 getaddrinfo()를 호출할 수 있습니다!

물론 여기서부터 재미가 시작됩니다. res는 이제 struct addrinfo 연결 리스트를 가리키고, 여러분은 이 리스트를 순회하면서 hints로 전달한 조건에 맞는 주소를 모두 얻을 수 있습니다.

다만 여러 이유로 동작하지 않는 주소가 섞여 있을 수 있습니다. 그래서 Linux man 페이지의 예제는 리스트를 순회하면서 socket()과 connect()를 차례로 호출하고, AI_PASSIVE 플래그로 서버를 준비하는 경우에는 bind()를 호출해 성공할 때까지 시도합니다.

마지막으로 연결 리스트를 다 쓰고 나면 freeaddrinfo()를 호출해 메모리를 해제해야 합니다. 안 그러면 메모리 누수가 나고, 어떤 분들은 몹시 속상해할 겁니다.

반환값

성공하면 0을, 오류가 발생하면 0이 아닌 값을 반환합니다. 0이 아닌 값이 반환되면 gai_strerror() 함수를 사용해 그 반환값에 들어 있는 에러 코드를 출력 가능한 문자열로 바꿀 수 있습니다.

예제

// code for a client connecting to a server
// namely a stream socket to www.example.com on port 80 (http)
// either IPv4 or IPv6

int sockfd;  
struct addrinfo hints, *servinfo, *p;
int rv;

memset(&hints, 0, sizeof hints);
hints.ai_family = AF_UNSPEC; // use AF_INET6 to force IPv6
hints.ai_socktype = SOCK_STREAM;

rv = getaddrinfo("www.example.com", "http", &hints, &servinfo);
if (rv != 0) {
    fprintf(stderr, "getaddrinfo: %s\n", gai_strerror(rv));
    exit(1);
}

// loop through all the results and connect to the first we can
for(p = servinfo; p != NULL; p = p->ai_next) {
    if ((sockfd = socket(p->ai_family, p->ai_socktype,
            p->ai_protocol)) == -1) {
        perror("socket");
        continue;
    }

    if (connect(sockfd, p->ai_addr, p->ai_addrlen) == -1) {
        perror("connect");
        close(sockfd);
        continue;
    }

    break; // if we get here, we must have connected successfully
}

if (p == NULL) {
    // looped off the end of the list with no connection
    fprintf(stderr, "failed to connect\n");
    exit(2);
}

freeaddrinfo(servinfo); // all done with this structure
// code for a server waiting for connections
// namely a stream socket on port 3490, on this host's IP
// either IPv4 or IPv6.

int sockfd;  
struct addrinfo hints, *servinfo, *p;
int rv;

memset(&hints, 0, sizeof hints);
hints.ai_family = AF_UNSPEC; // use AF_INET6 to force IPv6
hints.ai_socktype = SOCK_STREAM;
hints.ai_flags = AI_PASSIVE; // use my IP address

if ((rv = getaddrinfo(NULL, "3490", &hints, &servinfo)) != 0) {
    fprintf(stderr, "getaddrinfo: %s\n", gai_strerror(rv));
    exit(1);
}

// loop through all the results and bind to the first we can
for(p = servinfo; p != NULL; p = p->ai_next) {
    if ((sockfd = socket(p->ai_family, p->ai_socktype,
            p->ai_protocol)) == -1) {
        perror("socket");
        continue;
    }

    if (bind(sockfd, p->ai_addr, p->ai_addrlen) == -1) {
        close(sockfd);
        perror("bind");
        continue;
    }

    break; // if we get here, we must have connected successfully
}

if (p == NULL) {
    // looped off the end of the list with no successful bind
    fprintf(stderr, "failed to bind socket\n");
    exit(2);
}

freeaddrinfo(servinfo); // all done with this structure

9.6 gethostname()

형식

#include <sys/unistd.h>

int gethostname(char *name, size_t len);

설명

여러분의 시스템에는 이름이 있습니다. 다들 그렇죠. 지금까지 이야기한 네트워크 관련 내용보다는 조금 더 유닉스스러운 주제이지만, 그래도 쓸모는 있습니다.

char hostname[128];

gethostname(hostname, sizeof hostname);
printf("My hostname: %s\n", hostname);

9.7 gethostbyname(), gethostbyaddr()

호스트 이름으로 IP 주소를 얻거나, 그 반대로 얻는다

형식

#include <sys/socket.h>
#include <netdb.h>

struct hostent *gethostbyname(const char *name); // DEPRECATED!
struct hostent *gethostbyaddr(const char *addr, int len, int type);

설명

주의: 이 두 함수는 getaddrinfo()와 getnameinfo()로 대체되었습니다! 특히 gethostbyname()은 IPv6와 잘 맞지 않습니다.

이 함수들은 호스트 이름과 IP 주소를 서로 대응시켜 줍니다. 예를 들어 “www.example.com”이 있다면, gethostbyname()으로 IP 주소를 얻어 struct in_addr에 저장할 수 있습니다.

반대로 struct in_addr나 struct in6_addr가 있다면 gethostbyaddr()로 다시 호스트 이름을 얻을 수 있습니다. gethostbyaddr()는 실제로 IPv6와 호환되지만, 대신 더 새롭고 번듯한 getnameinfo()를 사용하는 편이 좋습니다.

(점과 숫자로 된 형식의 IP 주소 문자열을 가지고 호스트 이름을 찾고 싶다면, AI_CANONNAME 플래그와 함께 getaddrinfo()를 쓰는 편이 더 낫습니다.)

gethostbyname()은 “www.yahoo.com” 같은 문자열을 받아, IP 주소를 포함한 많은 정보를 담고 있는 struct hostent를 반환합니다. 다른 정보로는 공식 호스트 이름, 별칭 목록, 주소 타입, 주소 길이, 주소 목록 등이 있습니다. 범용 구조체이지만, 어떻게 생겼는지만 보면 우리 용도에는 꽤 쉽게 사용할 수 있습니다.

gethostbyaddr()는 struct in_addr 또는 struct in6_addr를 받아 대응하는 호스트 이름(있다면)을 돌려주므로, 일종의 gethostbyname() 역방향이라 볼 수 있습니다. 파라미터를 보면 addr 타입이 char*로 되어 있어도, 실제로는 struct in_addr에 대한 포인터를 넘기고 싶을 것입니다. len은 sizeof(struct in_addr)이어야 하고, type은 AF_INET이어야 합니다.

그럼 반환되는 struct hostent는 무엇일까요? 여기에 대상 호스트에 관한 여러 정보가 들어 있습니다.

필드 설명
char *h_name 실제 canonical 호스트 이름입니다.
char **h_aliases 배열처럼 접근할 수 있는 별칭 목록입니다. 마지막 원소는 NULL입니다.
int h_addrtype 결과 주소 타입입니다. 우리 용도에서는 사실상 AF_INET이어야 합니다.
int length 주소 길이를 바이트 단위로 나타냅니다. IPv4 주소라면 4입니다.
char **h_addr_list 이 호스트의 IP 주소 목록입니다. 타입은 char**지만, 실제로는 struct in_addr* 배열이라고 보면 됩니다. 마지막 원소는 NULL입니다.
h_addr h_addr_list[0]에 대해 흔히 정의되는 별칭입니다. 이 호스트의 IP 주소 하나만 아무거나 필요하다면(맞습니다, 여러 개일 수도 있습니다) 이 필드를 쓰면 됩니다.

반환값

성공하면 결과 struct hostent에 대한 포인터를 반환하고, 오류가 발생하면 NULL을 반환합니다.

이 함수들은 일반적인 perror() 같은 오류 출력 방식 대신, h_errno 변수에 병행되는 결과를 남깁니다. 이 값은 herror() 또는 hstrerror() 함수로 출력할 수 있습니다. 동작 방식은 여러분이 익숙한 고전적인 errno, perror(), strerror()와 거의 같습니다.

예제

#include <errno.h>
#include <netdb.h>

int main(int argc, char *argv[])
{

    if (argc != 2) {
        fprintf(stderr,"usage: ghbn hostname\n");
        return 1;
    }

    if ((he = gethostbyname(argv[1])) == NULL) {  // get host info
        herror("gethostbyname");
        return 2;
    }

    // print information about this host:
    printf("Official name is: %s\n", he->h_name);
    printf("    IP addresses: ");
    addr_list = (struct in_addr **)he->h_addr_list;
    for(i = 0; addr_list[i] != NULL; i++) {
        printf("%s ", inet_ntoa(*addr_list[i]));
    }
    printf("\n");

    return 0;
}

9.8 getnameinfo()

주어진 struct sockaddr에 대해 호스트 이름과 서비스 이름 정보를 조회한다.

형식

#include <sys/socket.h>
#include <netdb.h>

int getnameinfo(const struct sockaddr *sa, socklen_t salen,
                char *host, size_t hostlen,
                char *serv, size_t servlen, int flags);

설명

이 함수는 getaddrinfo()의 반대 역할을 합니다. 즉, 이미 값이 채워져 있는 struct sockaddr를 받아 그에 대한 이름과 서비스 이름을 조회합니다. 예전의 gethostbyaddr()와 getservbyport()를 대체합니다.

sa 파라미터에는 struct sockaddr에 대한 포인터를 넘겨야 합니다. 실제로는 캐스팅된 struct sockaddr_in 또는 struct sockaddr_in6일 가능성이 큽니다. 그리고 salen에는 그 struct의 길이를 넣습니다.

조회 결과인 호스트 이름과 서비스 이름은 host 및 serv 파라미터가 가리키는 영역에 기록됩니다. 물론 이 버퍼들의 최대 길이도 hostlen과 servlen으로 지정해야 합니다.

마지막으로 여러 플래그를 넘길 수 있는데, 여기서는 유용한 것 두 가지만 보겠습니다. NI_NOFQDN은 host에 전체 도메인 이름이 아니라 호스트 이름만 들어가게 합니다. NI_NAMEREQD는 DNS 조회로 이름을 찾지 못하면 함수가 실패하게 만듭니다. 이 플래그를 주지 않았는데 이름을 찾지 못하면, getnameinfo()는 대신 IP 주소의 문자열 표현을 host에 넣습니다.

늘 그렇듯 자세한 내용은 로컬 man 페이지를 확인하세요.

반환값

성공하면 0을, 오류가 발생하면 0이 아닌 값을 반환합니다. 반환값이 0이 아니면 gai_strerror()에 넘겨 사람이 읽을 수 있는 문자열로 바꿀 수 있습니다. 자세한 내용은 getaddrinfo를 참고하세요.

예제

struct sockaddr_in6 sa; // could be IPv4 if you want
char host[1024];
char service[20];

// pretend sa is full of good information about the host and port...

getnameinfo(&sa, sizeof sa, host, sizeof host, service,
            sizeof service, 0);

printf("   host: %s\n", host);    // e.g. "www.example.com"
printf("service: %s\n", service); // e.g. "http"

9.9 getpeername()

연결된 원격 측의 주소 정보를 반환한다

형식

#include <sys/socket.h>

int getpeername(int s, struct sockaddr *addr, socklen_t *len);

설명

원격 연결을 accept()했거나, 서버에 connect()했다면 이제 여러분은 이른바 peer를 가지게 됩니다. peer란 단순히 여러분이 연결된 컴퓨터를 뜻하며, IP 주소와 포트로 식별됩니다. 그래서…

getpeername()은 여러분이 연결된 머신 정보가 채워진 struct sockaddr_in을 간단히 반환합니다.

왜 이름(name)이라고 부를까요? 우리가 이 가이드에서 쓰는 인터넷 소켓뿐 아니라 소켓에는 여러 종류가 있고, 그래서 “name”이라는 단어가 모든 경우를 포괄하는 무난한 일반 용어였기 때문입니다. 우리 경우 peer의 “이름”은 곧 IP 주소와 포트입니다.

함수는 결과 주소의 크기를 len에 돌려주지만, 호출하기 전에 len에 addr의 크기를 미리 넣어 두어야 합니다.

반환값

성공하면 0을, 오류가 발생하면 -1을 반환합니다. 이때 errno가 알맞게 설정됩니다.

예제

// assume s is a connected socket

socklen_t len;
struct sockaddr_storage addr;
char ipstr[INET6_ADDRSTRLEN];
int port;

len = sizeof addr;
getpeername(s, (struct sockaddr*)&addr, &len);

// deal with both IPv4 and IPv6:
if (addr.ss_family == AF_INET) {
    struct sockaddr_in *s = (struct sockaddr_in *)&addr;
    port = ntohs(s->sin_port);
    inet_ntop(AF_INET, &s->sin_addr, ipstr, sizeof ipstr);
} else { // AF_INET6
    struct sockaddr_in6 *s = (struct sockaddr_in6 *)&addr;
    port = ntohs(s->sin6_port);
    inet_ntop(AF_INET6, &s->sin6_addr, ipstr, sizeof ipstr);
}

printf("Peer IP address: %s\n", ipstr);
printf("Peer port      : %d\n", port);

9.10 errno

마지막 시스템 호출의 에러 코드를 담고 있다

형식

#include <errno.h>

int errno;

설명

많은 시스템 호출의 에러 정보를 담고 있는 변수가 바로 이것입니다. 기억하시겠지만 socket()이나 listen() 같은 함수는 오류가 나면 -1을 반환하고, 어떤 오류가 발생했는지를 정확히 알려 주기 위해 errno 값을 설정합니다.

헤더 파일 errno.h에는 EADDRINUSE, EPIPE, ECONNREFUSED 같은 에러용 상수 심볼 이름이 많이 정의되어 있습니다. 로컬 man 페이지를 보면 어떤 에러 코드가 반환될 수 있는지 알 수 있고, 실행 중 그 값을 이용해 서로 다른 오류를 서로 다르게 처리할 수 있습니다.

혹은 더 흔한 방식으로, perror()나 strerror()를 호출해 사람이 읽기 쉬운 오류 메시지를 얻을 수 있습니다.

멀티스레드를 좋아하는 분들을 위해 덧붙이면, 대부분의 시스템에서 errno는 스레드 안전한 방식으로 정의되어 있습니다. 즉 실제 전역 변수는 아니지만, 단일 스레드 환경의 전역 변수처럼 행동합니다.

반환값

이 변수의 값은 가장 최근에 발생한 오류를 나타냅니다. 마지막 동작이 성공했다면 “성공”에 해당하는 코드가 들어 있을 수도 있습니다.

예제

s = socket(PF_INET, SOCK_STREAM, 0);
if (s == -1) {
    perror("socket"); // or use strerror()
}

tryagain:
if (select(n, &readfds, NULL, NULL) == -1) {
    // an error has occurred!!

    // if we were only interrupted, just restart the select() call:
    if (errno == EINTR) goto tryagain;  // AAAA! goto!!!

    // otherwise it's a more serious error:
    perror("select");
    exit(1);
}

9.11 fcntl()

소켓 디스크립터를 제어한다

형식

#include <sys/unistd.h>
#include <sys/fcntl.h>

int fcntl(int s, int cmd, long arg);

설명

이 함수는 보통 파일 잠금이나 그 밖의 파일 지향 작업에 사용되지만, 가끔 보거나 직접 쓰게 될 소켓 관련 기능도 몇 가지 가지고 있습니다.

s 파라미터는 조작하려는 소켓 디스크립터이고, cmd는 F_SETFL로 설정해야 하며, arg에는 아래 명령 중 하나를 넣을 수 있습니다. (말했듯이 fcntl()에는 여기서 다루는 것보다 더 많은 기능이 있지만, 저는 소켓 관련 부분에만 집중하겠습니다.)

cmd 설명
O_NONBLOCK 소켓을 논블로킹 모드로 설정합니다. 자세한 내용은 블로킹 절을 참고하세요.
O_ASYNC 소켓이 비동기 I/O를 하도록 설정합니다. 소켓에서 recv()할 데이터가 준비되면 SIGIO 시그널이 발생합니다. 흔히 보는 기능은 아니고, 이 가이드 범위도 벗어납니다. 아마 일부 시스템에서만 지원되기도 할 겁니다.

반환값

성공하면 0을, 오류가 발생하면 -1을 반환합니다. 이때 errno가 알맞게 설정됩니다.

실제로는 fcntl() 시스템 호출의 사용 방식에 따라 반환값도 달라집니다. 하지만 여기서는 소켓 관련 부분이 아니어서 다루지 않았습니다. 자세한 내용은 로컬 fcntl() man 페이지를 보세요.

예제

int s = socket(PF_INET, SOCK_STREAM, 0);

fcntl(s, F_SETFL, O_NONBLOCK);  // set to non-blocking
fcntl(s, F_SETFL, O_ASYNC);     // set to asynchronous I/O

9.12 htons(), htonl(), ntohs(), ntohl()

멀티바이트 정수 타입을 호스트 바이트 순서에서 네트워크 바이트 순서로 변환한다

형식

#include <netinet/in.h>

uint32_t htonl(uint32_t hostlong);
uint16_t htons(uint16_t hostshort);
uint32_t ntohl(uint32_t netlong);
uint16_t ntohs(uint16_t netshort);

설명

여러분을 정말 행복하지 않게 만드는 사실 하나는, 서로 다른 컴퓨터가 멀티바이트 정수(char보다 큰 모든 정수)의 내부 바이트 순서를 서로 다르게 쓴다는 점입니다. 그래서 Intel 머신에서 2바이트 short int를 예전 Mac(그러니까 걔들도 Intel이 되기 전의 Mac)으로 send()하면, 한쪽이 숫자 1이라고 생각한 값을 다른 쪽은 256으로 해석할 수 있고, 그 반대도 마찬가지입니다.

이 문제를 피하는 방법은 모두가 차이를 잠시 접어 두고 Motorola와 IBM이 옳았고 Intel이 좀 이상한 방식으로 했다고 합의한 다음, 데이터를 보내기 전에 바이트 순서를 “big-endian”으로 바꾸는 것입니다. Intel은 “little-endian” 머신이므로, 우리가 선호하는 이 바이트 순서를 좀 더 정치적으로 올바르게 “네트워크 바이트 순서”라고 부릅니다. 그래서 이 함수들은 여러분의 네이티브 바이트 순서와 네트워크 바이트 순서 사이를 오가며 변환합니다.

(즉 Intel에서는 이 함수들이 바이트 순서를 전부 뒤집고, PowerPC에서는 이미 네트워크 바이트 순서이므로 아무 일도 하지 않습니다. 그래도 누군가 Intel 머신에서 빌드해도 잘 동작하게 하려면 코드에서는 항상 이 함수를 써야 합니다.)

여기서 다루는 타입은 32비트(4바이트, 아마 int)와 16비트(2바이트, 대개 short) 숫자라는 점에 유의하세요.

일부 시스템에는 비표준 64비트 변형도 있습니다. <endian.h>가 있다면 htobe64()50와 그 친척 함수들을 살펴보세요. GCC에는 128비트까지 지원하는 byte swapping built-ins51도 있습니다. 직접 만들어 써도52 되지만, 실제로 스왑은 little-endian 머신에서만 해야 합니다!

어쨌든 이 함수들의 이름은 먼저 무엇으로부터 변환하는지를 정하고 읽으면 됩니다. 호스트(즉 여러분 머신) 바이트 순서에서 변환한다면 함수 이름 첫 글자는 “h”입니다. 아니면 “network”의 “n”이 됩니다. 가운데는 항상 “to”이고, 끝에서 두 번째 글자는 무엇으로 변환하는지를 나타냅니다. 마지막 글자는 데이터 크기로, “s”는 short, “l”은 long입니다. 그래서 다음과 같습니다.

함수 설명
htons() 호스트(h)에서 네트워크(n) short(s)로
htonl() 호스트(h)에서 네트워크(n) long(l)으로
ntohs() 네트워크(n)에서 호스트(h) short(s)로
ntohl() 네트워크(n)에서 호스트(h) long(l)으로

반환값

각 함수는 변환된 값을 반환합니다.

예제

uint32_t some_long = 10;
uint16_t some_short = 20;

uint32_t network_byte_order;

// convert and send
network_byte_order = htonl(some_long);
send(s, &network_byte_order, sizeof(uint32_t), 0);

some_short == ntohs(htons(some_short)); // this expression is true

9.13 inet_ntoa(), inet_aton(), inet_addr

점과 숫자 형식의 IP 주소 문자열과 struct in_addr 사이를 상호 변환한다

형식

#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>

// ALL THESE ARE DEPRECATED!
// Use inet_pton() or inet_ntop() instead!

char *inet_ntoa(struct in_addr in);
int inet_aton(const char *cp, struct in_addr *inp);
in_addr_t inet_addr(const char *cp);

설명

이 함수들은 IPv6를 다루지 못하므로 deprecated 상태입니다! 대신 inet_ntop()이나 inet_pton()을 사용하세요! 그래도 여전히 현장에서 마주칠 수 있어서 여기 포함했습니다.

이 함수들은 모두 struct in_addr(아마 여러분의 struct sockaddr_in 일부일 가능성이 큽니다)와 점과 숫자 형식 문자열(예: “192.168.5.10”) 사이를 양방향으로 변환합니다. 명령줄 등으로 전달된 IP 주소가 있다면, connect() 등에 넘길 struct in_addr를 얻는 가장 쉬운 방법입니다. 더 강력한 게 필요하다면 gethostbyname() 같은 DNS 함수를 써 보거나, 아니면 여러분 나라에서 쿠데타를 시도해 보세요.

inet_ntoa()는 struct in_addr 안의 네트워크 주소를 점과 숫자 형식 문자열로 변환합니다. “ntoa”의 “n”은 network를, “a”는 역사적 이유로 ASCII를 뜻합니다. 즉 “Network To ASCII”입니다. 참고로 “toa”에 대응하는 친구로 C 라이브러리의 atoi()가 있는데, 이건 ASCII 문자열을 정수로 바꿉니다.

inet_aton()은 그 반대로, 점과 숫자 형식 문자열을 in_addr_t로 변환합니다. 이 타입은 여러분 struct in_addr 안의 s_addr 필드 타입입니다.

마지막으로 inet_addr()는 사실상 inet_aton()과 거의 같은 일을 하는 더 오래된 함수입니다. 이론적으로는 deprecated지만 꽤 자주 보게 될 것이고, 쓴다고 해서 경찰이 잡으러 오지는 않습니다.

반환값

inet_aton()은 주소가 유효하면 0이 아닌 값을 반환하고, 유효하지 않으면 0을 반환합니다.

inet_ntoa()는 점과 숫자 형식 문자열을 static 버퍼에 담아 반환하며, 이 버퍼는 함수가 호출될 때마다 덮어써집니다.

inet_addr()는 주소를 in_addr_t로 반환하고, 오류가 있으면 -1을 반환합니다. (그런데 이것은 유효한 IP 주소인 “255.255.255.255”를 변환했을 때와 같은 결과입니다. 그래서 inet_aton()이 더 낫습니다.)

예제

struct sockaddr_in antelope;
char *some_addr;

inet_aton("10.0.0.1", &antelope.sin_addr); // store IP in antelope

some_addr = inet_ntoa(antelope.sin_addr); // return the IP
printf("%s\n", some_addr); // prints "10.0.0.1"

// and this call is the same as the inet_aton() call, above:
antelope.sin_addr.s_addr = inet_addr("10.0.0.1");

9.14 inet_ntop(), inet_pton()

IP 주소를 사람이 읽을 수 있는 형식으로 바꾸고, 다시 되돌린다.

형식

#include <arpa/inet.h>

const char *inet_ntop(int af, const void *src,
                      char *dst, socklen_t size);

int inet_pton(int af, const char *src, void *dst);

설명

이 함수들은 사람이 읽을 수 있는 IP 주소를 다루고, 이를 다양한 함수와 시스템 호출에서 사용할 수 있는 바이너리 표현으로 바꾸는 데 쓰입니다. “n”은 “network”, “p”는 “presentation” 또는 “text presentation”을 뜻합니다. 그냥 “printable”이라고 생각해도 됩니다. “ntop”은 “network to printable”입니다. 보이시죠?

IP 주소를 볼 때 바이너리 숫자 덩어리를 보고 싶지 않을 때가 있습니다. 192.0.2.180이나 2001:db8:8714:3a90::12처럼 보기 좋은 문자열이 필요하죠. 그런 경우 inet_ntop()이 딱입니다.

inet_ntop()은 af 파라미터로 주소 계열(AF_INET 또는 AF_INET6)을 받습니다. src 파라미터는 문자열로 바꾸고 싶은 주소가 들어 있는 struct in_addr 또는 struct in6_addr 포인터여야 합니다. 마지막으로 dst와 size는 목적지 문자열 포인터와 그 문자열의 최대 길이입니다.

dst 문자열의 최대 길이는 얼마로 잡아야 할까요? IPv4와 IPv6 주소의 최대 길이는 얼마일까요? 다행히 이를 도와주는 매크로가 몇 개 있습니다. 최대 길이는 각각 INET_ADDRSTRLEN과 INET6_ADDRSTRLEN입니다.

반대로, 사람이 읽을 수 있는 IP 주소 문자열을 가지고 이를 struct sockaddr_in이나 struct sockaddr_in6에 채워 넣고 싶을 수도 있습니다. 그럴 때는 반대 함수인 inet_pton()을 사용합니다.

inet_pton() 역시 af 파라미터로 주소 계열(AF_INET 또는 AF_INET6)을 받습니다. src는 출력 가능한 형태의 IP 주소 문자열을 가리키는 포인터입니다. 마지막 dst는 결과를 저장할 위치를 가리키며, 보통 struct in_addr 또는 struct in6_addr일 것입니다.

이 함수들은 DNS 조회는 하지 않습니다. 그건 getaddrinfo()가 필요합니다.

반환값

inet_ntop()은 성공하면 dst 파라미터를 반환하고, 실패하면 NULL을 반환합니다. 이때 errno가 설정됩니다.

inet_pton()은 성공하면 1을 반환합니다. 오류가 나면 -1을 반환하며(errno 설정), 입력이 유효한 IP 주소가 아니면 0을 반환합니다.

예제

// IPv4 demo of inet_ntop() and inet_pton()

struct sockaddr_in sa;
char str[INET_ADDRSTRLEN];

// store this IP address in sa:
inet_pton(AF_INET, "192.0.2.33", &(sa.sin_addr));

// now get it back and print it
inet_ntop(AF_INET, &(sa.sin_addr), str, INET_ADDRSTRLEN);

printf("%s\n", str); // prints "192.0.2.33"
// IPv6 demo of inet_ntop() and inet_pton()
// (basically the same except with a bunch of 6s thrown around)

struct sockaddr_in6 sa;
char str[INET6_ADDRSTRLEN];

// store this IP address in sa:
inet_pton(AF_INET6, "2001:db8:8714:3a90::12", &(sa.sin6_addr));

// now get it back and print it
inet_ntop(AF_INET6, &(sa.sin6_addr), str, INET6_ADDRSTRLEN);

printf("%s\n", str); // prints "2001:db8:8714:3a90::12"
// Helper function you can use:

//Convert a struct sockaddr address to a string, IPv4 and IPv6:

char *get_ip_str(const struct sockaddr *sa, char *s, size_t maxlen)
{
    switch(sa->sa_family) {
        case AF_INET:
            inet_ntop(AF_INET,
                    &(((struct sockaddr_in *)sa)->sin_addr), s,
                    maxlen);
            break;

        case AF_INET6:
            inet_ntop(AF_INET6,
                    &(((struct sockaddr_in6 *)sa)->sin6_addr), s,
                    maxlen);
            break;

        default:
            strncpy(s, "Unknown AF", maxlen);
            return NULL;
    }

    return s;
}

9.15 listen()

소켓이 들어오는 연결을 기다리도록 만든다

형식

#include <sys/socket.h>

int listen(int s, int backlog);

설명

socket() 시스템 호출로 만든 소켓 디스크립터를 가져다가, 들어오는 연결을 기다리도록 설정할 수 있습니다. 바로 이 점이 서버와 클라이언트를 갈라놓는 요소입니다.

backlog 파라미터는 시스템에 따라 조금씩 의미가 다르지만, 대체로 커널이 새 연결을 거부하기 전에 대기시켜 둘 수 있는 보류 연결 수를 뜻합니다. 따라서 새 연결이 들어오면 backlog가 꽉 차지 않도록 재빨리 accept()해야 합니다. 일단 10 정도로 두고, 부하가 높을 때 클라이언트가 “Connection refused”를 받기 시작하면 더 크게 올려 보세요.

listen()을 호출하기 전에 서버는 특정 포트 번호에 자신을 연결하기 위해 bind()를 호출해야 합니다. 클라이언트는 서버 IP 주소의 그 포트 번호로 연결합니다.

반환값

성공하면 0을, 오류가 발생하면 -1을 반환합니다. 이때 errno가 알맞게 설정됩니다.

예제

struct addrinfo hints, *res;
int sockfd;

// first, load up address structs with getaddrinfo():

memset(&hints, 0, sizeof hints);
hints.ai_family = AF_UNSPEC;  // use IPv4 or IPv6, whichever
hints.ai_socktype = SOCK_STREAM;
hints.ai_flags = AI_PASSIVE;     // fill in my IP for me

getaddrinfo(NULL, "3490", &hints, &res);

// make a socket:

sockfd = socket(res->ai_family, res->ai_socktype,
    res->ai_protocol);

// bind it to the port we passed in to getaddrinfo():

bind(sockfd, res->ai_addr, res->ai_addrlen);

listen(sockfd, 10); // set sockfd up to be a server socket

// then have an accept() loop down here somewhere

9.16 perror(), strerror()

오류를 사람이 읽을 수 있는 문자열로 출력한다

형식

#include <stdio.h>
#include <string.h>   // for strerror()

void perror(const char *s);
char *strerror(int errnum);

설명

워낙 많은 함수가 오류 시 -1을 반환하고 errno 변수에 어떤 숫자를 넣어 두기 때문에, 그 값을 사람이 이해할 수 있는 형태로 쉽게 출력할 수 있으면 참 좋겠죠.

다행히 perror()가 그 일을 해 줍니다. 오류 메시지 앞에 추가 설명을 붙이고 싶다면 s 파라미터가 그 문자열을 가리키게 하면 됩니다. (s를 NULL로 두면 아무 추가 문구도 출력되지 않습니다.)

한마디로 이 함수는 ECONNRESET 같은 errno 값을 받아 “Connection reset by peer.”처럼 보기 좋게 출력합니다.

strerror()는 perror()와 매우 비슷하지만, 주어진 값(보통 errno)에 대한 에러 메시지 문자열 포인터를 반환한다는 점이 다릅니다.

반환값

strerror()는 에러 메시지 문자열에 대한 포인터를 반환합니다.

예제

int s;

s = socket(PF_INET, SOCK_STREAM, 0);

if (s == -1) { // some error has occurred
    // prints "socket error: " + the error message:
    perror("socket error");
}

// similarly:
if (listen(s, 10) == -1) {
    // this prints "an error: " + the error message from errno:
    printf("an error: %s\n", strerror(errno));
}

9.17 poll()

여러 소켓에서 발생한 이벤트를 동시에 검사한다

형식

#include <sys/poll.h>

int poll(struct pollfd *ufds, unsigned int nfds, int timeout);

설명

이 함수는 여러 파일 디스크립터 집합을 감시해 이벤트를 본다는 점에서 select()와 매우 비슷합니다. 예를 들어 recv()할 준비가 된 데이터, send() 가능한 소켓, recv()할 준비가 된 out-of-band 데이터, 오류 등을 감시할 수 있습니다.

기본 아이디어는 ufds에 nfds개의 struct pollfd 배열을 넘기고, 밀리초 단위의 타임아웃도 함께 넘기는 것입니다. (1초는 1000밀리초입니다.) 영원히 기다리고 싶다면 timeout에 음수를 넣을 수 있습니다. 타임아웃 시점까지 어느 소켓 디스크립터에서도 이벤트가 일어나지 않으면 poll()이 반환합니다.

struct pollfd 배열의 각 원소는 하나의 소켓 디스크립터를 나타내며, 다음 필드를 가집니다.

struct pollfd {
    int fd;         // the socket descriptor
    short events;   // bitmap of events we're interested in
    short revents;  // after return, bitmap of events that occurred
};

poll()을 호출하기 전에 fd에 소켓 디스크립터를 넣습니다. (fd를 음수로 설정하면 해당 struct pollfd는 무시되고, revents 필드는 0으로 설정됩니다.) 그리고 아래 매크로를 비트 OR 하여 events 필드를 구성합니다.

매크로 설명
POLLIN 이 소켓에서 recv()할 데이터가 준비되면 알려 줍니다.
POLLOUT 이 소켓에 블록 없이 send()할 수 있게 되면 알려 줍니다.
POLLPRI 이 소켓에서 out-of-band 데이터를 recv()할 수 있게 되면 알려 줍니다.

poll() 호출이 반환되고 나면 revents 필드가 위 필드들의 비트 OR 형태로 채워져, 어떤 디스크립터에서 실제로 어떤 이벤트가 일어났는지 알려 줍니다. 추가로 다음 필드들도 나타날 수 있습니다.

매크로 설명
POLLERR 이 소켓에서 오류가 발생했습니다.
POLLHUP 연결 반대편이 연결을 끊었습니다.
POLLNVAL 소켓 디스크립터 fd에 뭔가 문제가 있습니다. 초기화되지 않았을 수도 있습니다.

반환값

이벤트가 발생한 ufds 배열 원소 수를 반환합니다. 타임아웃이 발생한 경우 이 값은 0일 수 있습니다. 오류 시에는 -1을 반환하며, errno가 알맞게 설정됩니다.

예제

int s1, s2;
int rv;
char buf1[256], buf2[256];
struct pollfd ufds[2];

s1 = socket(PF_INET, SOCK_STREAM, 0);
s2 = socket(PF_INET, SOCK_STREAM, 0);

// pretend we've connected both to a server at this point
//connect(s1, ...)...
//connect(s2, ...)...

// set up the array of file descriptors.
//
// in this example, we want to know when there's normal or
// out-of-band (OOB) data ready to be recv()'d...

ufds[0].fd = s1;
ufds[0].events = POLLIN | POLLPRI; // check for normal or OOB

ufds[1].fd = s2;
ufds[1].events = POLLIN; // check for just normal data

// wait for events on the sockets, 3.5 second timeout
rv = poll(ufds, 2, 3500);

if (rv == -1) {
    perror("poll"); // error occurred in poll()
} else if (rv == 0) {
    printf("Timeout occurred! No data after 3.5 seconds.\n");
} else {
    // check for events on s1:
    if (ufds[0].revents & POLLIN) {
        recv(s1, buf1, sizeof buf1, 0); // receive normal data
    }
    if (ufds[0].revents & POLLPRI) {
        recv(s1, buf1, sizeof buf1, MSG_OOB); // out-of-band data
    }

    // check for events on s2:
    if (ufds[1].revents & POLLIN) {
        recv(s1, buf2, sizeof buf2, 0);
    }
}

9.18 recv(), recvfrom()

소켓에서 데이터를 받는다

형식

#include <sys/types.h>
#include <sys/socket.h>

ssize_t recv(int s, void *buf, size_t len, int flags);
ssize_t recvfrom(int s, void *buf, size_t len, int flags,
                 struct sockaddr *from, socklen_t *fromlen);

설명

소켓을 만들고 연결까지 마쳤다면, 원격 측에서 들어오는 데이터를 recv()(TCP SOCK_STREAM 소켓용) 또는 recvfrom()(UDP SOCK_DGRAM 소켓용)으로 읽을 수 있습니다.

두 함수 모두 소켓 디스크립터 s, 버퍼 포인터 buf, 버퍼 크기(바이트 단위) len, 그리고 함수 동작을 제어하는 flags 집합을 받습니다.

추가로 recvfrom()은 데이터가 어디서 왔는지를 알려 줄 struct sockaddr*인 from을 받으며, fromlen에는 struct sockaddr의 크기를 채웁니다. (호출 전에 fromlen도 from 또는 struct sockaddr의 크기로 초기화해야 합니다.)

그럼 이 함수에 어떤 멋진 플래그를 넘길 수 있을까요? 여기 몇 가지만 소개하지만, 더 많은 정보와 실제 시스템 지원 여부는 로컬 man 페이지를 확인하세요. 플래그들은 비트 OR 해서 함께 쓸 수 있고, 평범한 기본 recv()가 필요하다면 flags를 그냥 0으로 두면 됩니다.

매크로 설명
MSG_OOB Out of Band 데이터를 받습니다. send()에서 MSG_OOB 플래그를 붙여 보낸 데이터를 이렇게 받을 수 있습니다. 수신 측에서는 긴급 데이터가 있다는 뜻으로 SIGURG 시그널을 받게 되고, 그 시그널 핸들러 안에서 이 MSG_OOB 플래그와 함께 recv()를 호출할 수 있습니다.
MSG_PEEK recv()를 “살짝 보기만” 하고 싶다면 이 플래그를 사용합니다. 그러면 나중에 진짜 recv()(즉 MSG_PEEK 없이 호출하는 경우)를 했을 때 버퍼에 무엇이 기다리고 있는지 알 수 있습니다. 다음 recv() 호출을 미리 엿보는 것과 비슷합니다.
MSG_WAITALL len 파라미터로 지정한 데이터를 전부 받을 때까지 recv()가 반환하지 않게 합니다. 다만 시그널로 호출이 끊기거나 오류가 나거나 원격 측이 연결을 닫는 극단적인 상황에서는 여러분의 뜻을 무시할 수도 있습니다. 너무 화내지는 마세요.

recv()를 호출하면 읽을 데이터가 생길 때까지 블록됩니다. 블록되지 않게 하려면 소켓을 논블로킹으로 설정하거나, recv() 또는 recvfrom()를 호출하기 전에 select()나 poll()로 들어오는 데이터가 있는지 확인하세요.

반환값

실제로 받은 바이트 수를 반환합니다. 이 값은 len 파라미터에서 요청한 수보다 적을 수도 있습니다. 오류 시에는 -1을 반환하며, errno가 알맞게 설정됩니다.

원격 측이 연결을 닫았다면 recv()는 0을 반환합니다. 이것이 원격 측이 연결을 닫았는지 판별하는 정상적인 방법입니다. 정상적인 게 좋죠, 반항아 여러분!

예제

// stream sockets and recv()

struct addrinfo hints, *res;
int sockfd;
char buf[512];
int byte_count;

// get host info, make socket, and connect it
memset(&hints, 0, sizeof hints);
hints.ai_family = AF_UNSPEC;  // use IPv4 or IPv6, whichever
hints.ai_socktype = SOCK_STREAM;
getaddrinfo("www.example.com", "3490", &hints, &res);
sockfd = socket(res->ai_family, res->ai_socktype, res->ai_protocol);
connect(sockfd, res->ai_addr, res->ai_addrlen);

// all right! now that we're connected, we can receive some data!
byte_count = recv(sockfd, buf, sizeof buf, 0);
printf("recv()'d %d bytes of data in buf\n", byte_count);
// datagram sockets and recvfrom()

struct addrinfo hints, *res;
int sockfd;
int byte_count;
socklen_t fromlen;
struct sockaddr_storage addr;
char buf[512];
char ipstr[INET6_ADDRSTRLEN];

// get host info, make socket, bind it to port 4950
memset(&hints, 0, sizeof hints);
hints.ai_family = AF_UNSPEC;  // use IPv4 or IPv6, whichever
hints.ai_socktype = SOCK_DGRAM;
hints.ai_flags = AI_PASSIVE;
getaddrinfo(NULL, "4950", &hints, &res);
sockfd = socket(res->ai_family, res->ai_socktype, res->ai_protocol);
bind(sockfd, res->ai_addr, res->ai_addrlen);

// no need to accept(), just recvfrom():

fromlen = sizeof addr;
byte_count = recvfrom(sockfd, buf, sizeof buf, 0, &addr, &fromlen);

printf("recv()'d %d bytes of data in buf\n", byte_count);
printf("from IP address %s\n",
    inet_ntop(addr.ss_family,
        addr.ss_family == AF_INET?
            ((struct sockaddr_in *)&addr)->sin_addr:
            ((struct sockaddr_in6 *)&addr)->sin6_addr,
        ipstr, sizeof ipstr);

9.19 select()

소켓 디스크립터가 읽기/쓰기를 할 준비가 되었는지 확인한다

형식

#include <sys/select.h>

int select(int n, fd_set *readfds, fd_set *writefds,
           fd_set *exceptfds, struct timeval *timeout);

FD_SET(int fd, fd_set *set);
FD_CLR(int fd, fd_set *set);
FD_ISSET(int fd, fd_set *set);
FD_ZERO(fd_set *set);

설명

select() 함수는 여러 소켓을 동시에 검사해, recv()할 데이터가 대기 중인지, 블록 없이 send()할 수 있는지, 혹은 어떤 예외 상황이 발생했는지를 알아볼 수 있게 해 줍니다.

위의 FD_SET() 같은 매크로를 사용해 소켓 디스크립터 집합을 채웁니다. 집합이 준비되면 함수에 다음 파라미터 중 하나로 넘깁니다. 집합 안의 어떤 소켓이든 recv()할 준비가 되었는지 알고 싶다면 readfds, 어떤 소켓이든 send()할 준비가 되었는지 알고 싶다면 writefds, 그리고 어떤 소켓에서든 예외(오류)가 발생했는지 알고 싶다면 exceptfds를 사용합니다. 이 이벤트 타입에 관심이 없다면 각 파라미터는 NULL이어도 됩니다. select()가 반환한 뒤에는 집합의 값이 바뀌어서, 어떤 소켓이 읽기/쓰기를 할 준비가 되었고 어떤 소켓에서 예외가 났는지를 보여 줍니다.

첫 번째 파라미터 n은 가장 큰 번호의 소켓 디스크립터에 1을 더한 값입니다. (소켓 디스크립터도 그냥 int라는 점을 기억하세요.)

마지막의 struct timeval인 timeout은 select()가 이 집합들을 얼마 동안 검사할지를 정합니다. 타임아웃이 지나거나 이벤트가 발생하면, 둘 중 먼저 일어난 시점에 반환합니다. struct timeval에는 두 필드가 있습니다. tv_sec는 초 단위 시간이고, 여기에 마이크로초 단위의 tv_usec가 더해집니다. (1초는 1,000,000마이크로초입니다.)

보조 매크로들은 다음과 같은 일을 합니다.

매크로 설명
FD_SET(int fd, fd_set *set); fd를 set에 추가합니다.
FD_CLR(int fd, fd_set *set); fd를 set에서 제거합니다.
FD_ISSET(int fd, fd_set *set); fd가 set에 들어 있으면 참을 반환합니다.
FD_ZERO(fd_set *set); set의 모든 항목을 비웁니다.

Linux 사용자에게 드리는 참고: Linux의 select()는 “읽을 준비가 됨”을 반환해 놓고도 실제로는 읽을 준비가 안 된 상태일 수 있고, 그 결과 이어지는 read() 호출이 블록될 수 있습니다. 이 버그는 수신 소켓에 O_NONBLOCK 플래그를 설정해 EWOULDBLOCK 오류가 나게 한 뒤, 그 오류가 발생하면 무시하는 방식으로 우회할 수 있습니다. 소켓을 논블로킹으로 설정하는 방법은 fcntl() man 페이지를 참고하세요.

반환값

성공하면 집합 안에서 준비된 디스크립터 수를 반환하고, 타임아웃이 발생하면 0을, 오류가 나면 -1을 반환합니다. 이때 errno가 알맞게 설정됩니다. 또한 집합 자체도 어떤 소켓이 준비되었는지를 나타내도록 수정됩니다.

예제

int s1, s2, n;
fd_set readfds;
struct timeval tv;
char buf1[256], buf2[256];

// pretend we've connected both to a server at this point
//s1 = socket(...);
//s2 = socket(...);
//connect(s1, ...)...
//connect(s2, ...)...

// clear the set ahead of time
FD_ZERO(&readfds);

// add our descriptors to the set
FD_SET(s1, &readfds);
FD_SET(s2, &readfds);

// since we got s2 second, it's the "greater", so we use that for
// the n param in select()
n = s2 + 1;

// wait until either socket has data ready to be recv()d
// (timeout 10.5 secs)
tv.tv_sec = 10;
tv.tv_usec = 500000;
rv = select(n, &readfds, NULL, NULL, &tv);

if (rv == -1) {
    perror("select"); // error occurred in select()
} else if (rv == 0) {
    printf("Timeout occurred! No data after 10.5 seconds.\n");
} else {
    // one or both of the descriptors have data
    if (FD_ISSET(s1, &readfds)) {
        recv(s1, buf1, sizeof buf1, 0);
    }
    if (FD_ISSET(s2, &readfds)) {
        recv(s2, buf2, sizeof buf2, 0);
    }
}

9.20 setsockopt(), getsockopt()

소켓의 여러 옵션을 설정한다

형식

#include <sys/types.h>
#include <sys/socket.h>

int getsockopt(int s, int level, int optname, void *optval,
               socklen_t *optlen);
int setsockopt(int s, int level, int optname, const void *optval,
               socklen_t optlen);

설명

소켓은 꽤나 설정 가능한 녀석들입니다. 사실 너무 설정 가능한 나머지, 여기서 전부 다루지는 않겠습니다. 어차피 시스템 의존적인 부분도 많습니다. 대신 기본은 이야기하겠습니다.

말 그대로 이 함수들은 소켓의 특정 옵션을 읽고 설정합니다. Linux에서는 소켓 관련 정보가 모두 섹션 7의 socket man 페이지에 있습니다. (“man 7 socket”를 입력해 온갖 좋은 내용을 볼 수 있습니다.)

파라미터를 보면, s는 대상 소켓이고, level은 SOL_SOCKET으로 설정해야 합니다. 그리고 optname에 관심 있는 옵션 이름을 넣습니다. 모든 옵션은 역시 로컬 man 페이지를 봐야 하지만, 여기서는 그중 재미있는 것 몇 가지를 보겠습니다.

optname 설명
SO_BINDTODEVICE bind()로 IP 주소에 묶는 대신, 이 소켓을 eth0 같은 심볼릭 장치 이름에 바인드합니다. 장치 이름은 유닉스에서 ifconfig 명령으로 볼 수 있습니다.
SO_REUSEADDR 이미 그 포트에 활성 리스닝 소켓이 바인드되어 있지 않은 한, 다른 소켓도 이 포트에 bind()할 수 있게 합니다. 서버가 크래시한 뒤 재시작할 때 나오는 “Address already in use” 오류를 피하는 데 유용합니다.
SO_BROADCAST UDP 데이터그램(SOCK_DGRAM) 소켓이 브로드캐스트 주소로 보내고 받는 패킷을 다룰 수 있게 합니다. TCP 스트림 소켓에는 아무 일도 하지 않습니다. 정말로, 아무 일도요!!

optval 파라미터는 보통 해당 값을 나타내는 int에 대한 포인터입니다. 불리언의 경우 0은 거짓, 0이 아닌 값은 참입니다. 물론 여러분 시스템이 다르게 정의해 두지 않았다면 말이죠. 넘길 파라미터가 없다면 optval은 NULL이어도 됩니다.

마지막 파라미터 optlen은 optval의 길이로 설정해야 하며, 보통 sizeof(int)이지만 옵션에 따라 달라집니다. getsockopt()의 경우 이 값은 socklen_t 포인터이며, optval에 저장될 객체의 최대 크기를 지정해 버퍼 오버플로를 막습니다. 그리고 getsockopt()는 실제로 설정된 바이트 수를 반영하도록 optlen 값을 수정합니다.

경고: 일부 시스템(특히 Sun과 Windows)에서는 옵션 값이 int가 아니라 char일 수 있고, 예를 들어 정수 1 대신 문자 값 '1'로 설정되기도 합니다. 다시 말하지만 “man setsockopt”와 “man 7 socket”로 각자 시스템의 man 페이지를 꼭 확인하세요!

반환값

성공하면 0을, 오류가 발생하면 -1을 반환합니다. 이때 errno가 알맞게 설정됩니다.

예제

int optval;
int optlen;
char *optval2;

// set SO_REUSEADDR on a socket to true (1):
optval = 1;
setsockopt(s1, SOL_SOCKET, SO_REUSEADDR, &optval, sizeof optval);

// bind a socket to a device name (might not work on all systems):
optval2 = "eth1"; // 4 bytes long, so 4, below:
setsockopt(s2, SOL_SOCKET, SO_BINDTODEVICE, optval2, 4);

// see if the SO_BROADCAST flag is set:
getsockopt(s3, SOL_SOCKET, SO_BROADCAST, &optval, &optlen);
if (optval != 0) {
    print("SO_BROADCAST enabled on s3!\n");
}

9.21 send(), sendto()

소켓을 통해 데이터를 보낸다

형식

#include <sys/types.h>
#include <sys/socket.h>

ssize_t send(int s, const void *buf, size_t len, int flags);
ssize_t sendto(int s, const void *buf, size_t len,
               int flags, const struct sockaddr *to,
               socklen_t tolen);

설명

이 함수들은 소켓으로 데이터를 보냅니다. 일반적으로 send()는 TCP SOCK_STREAM 연결 소켓에, sendto()는 UDP SOCK_DGRAM 비연결 데이터그램 소켓에 사용됩니다. 비연결 소켓에서는 패킷을 보낼 때마다 목적지를 매번 지정해야 하므로, sendto()의 마지막 파라미터들이 패킷이 어디로 가는지를 정의합니다.

send()와 sendto() 모두에서 s는 소켓, buf는 보내려는 데이터 포인터, len은 보내려는 바이트 수, flags는 데이터 전송 방식을 추가로 지정하는 값입니다. 그냥 “일반적인” 데이터를 보내고 싶다면 flags를 0으로 두면 됩니다. 흔히 쓰는 플래그 몇 가지를 아래에 적었지만, 자세한 내용은 로컬 send() man 페이지를 보세요.

매크로 설명
MSG_OOB “Out of band” 데이터로 보냅니다. TCP가 이를 지원하며, 수신 시스템에 이 데이터가 일반 데이터보다 우선순위가 높다고 알리는 방식입니다. 수신 측은 SIGURG 시그널을 받게 되고, 큐에 있는 일반 데이터를 모두 먼저 받지 않아도 이 데이터를 받아 볼 수 있습니다.
MSG_DONTROUTE 이 데이터를 라우터를 거쳐 보내지 말고 로컬에만 머무르게 합니다.
MSG_DONTWAIT 송신 트래픽이 막혀 send()가 블록될 상황이면 EAGAIN을 반환하게 합니다. “이번 send에 한해서만 논블로킹을 켠다”는 느낌입니다. 자세한 내용은 블로킹 절을 참고하세요.
MSG_NOSIGNAL 더 이상 recv()하지 않는 원격 호스트에 send()하면 보통 SIGPIPE 시그널을 받게 됩니다. 이 플래그를 추가하면 그 시그널이 발생하지 않습니다.

반환값

실제로 전송된 바이트 수를 반환하고, 오류가 나면 -1을 반환합니다. 이때 errno가 알맞게 설정됩니다. 실제로 보낸 바이트 수는 요청한 수보다 적을 수 있다는 점에 유의하세요! 이를 다루는 보조 함수는 부분 send() 처리 절을 참고하세요.

또한 소켓이 어느 한쪽에서라도 닫힌 상태라면, send()를 호출한 프로세스는 SIGPIPE 시그널을 받습니다. (MSG_NOSIGNAL 플래그와 함께 send()를 호출한 경우는 예외입니다.)

예제

int spatula_count = 3490;
char *secret_message = "The Cheese is in The Toaster";

int stream_socket, dgram_socket;
struct sockaddr_in dest;
int temp;

// first with TCP stream sockets:

// assume sockets are made and connected
//stream_socket = socket(...
//connect(stream_socket, ...

// convert to network byte order
temp = htonl(spatula_count);
// send data normally:
send(stream_socket, &temp, sizeof temp, 0);

// send secret message out of band:
send(stream_socket, secret_message, strlen(secret_message)+1,
        MSG_OOB);

// now with UDP datagram sockets:
//getaddrinfo(...
//dest = ... // assume "dest" holds the address of the destination
//dgram_socket = socket(...

// send secret message normally:
sendto(dgram_socket, secret_message, strlen(secret_message)+1, 0, 
       (struct sockaddr*)&dest, sizeof dest);

9.22 shutdown()

소켓에서 더 이상의 송수신을 중단한다

형식

#include <sys/socket.h>

int shutdown(int s, int how);

설명

이제 됐다! 더는 못 참겠다! 이 소켓에서는 더 이상 send()는 못 하게 하고 싶지만, 그래도 recv()는 계속하고 싶다! 혹은 그 반대다! 이걸 어떻게 할까요?

소켓 디스크립터에 close()를 호출하면 읽기와 쓰기 양쪽이 모두 닫히고, 소켓 디스크립터도 해제됩니다. 한쪽만 닫고 싶다면 이 shutdown() 호출을 사용하면 됩니다.

파라미터를 보면 s는 당연히 이 동작을 수행할 소켓이고, 정확히 어떤 동작을 할지는 how 파라미터로 지정합니다. how는 더 이상의 recv()를 막는 SHUT_RD, 더 이상의 send()를 금지하는 SHUT_WR, 또는 둘 다 하는 SHUT_RDWR가 될 수 있습니다.

shutdown()은 소켓 디스크립터 자체를 해제하지는 않으므로, 완전히 종료했더라도 결국에는 close()를 호출해야 합니다.

자주 쓰이는 시스템 호출은 아닙니다.

반환값

성공하면 0을, 오류가 발생하면 -1을 반환합니다. 이때 errno가 알맞게 설정됩니다.

예제

int s = socket(PF_INET, SOCK_STREAM, 0);

// ...do some send()s and stuff in here...

// and now that we're done, don't allow any more sends()s:
shutdown(s, SHUT_WR);

9.23 socket()

소켓 디스크립터를 할당한다

형식

#include <sys/types.h>
#include <sys/socket.h>

int socket(int domain, int type, int protocol);

설명

소켓 관련 일을 하는 데 쓸 수 있는 새 소켓 디스크립터를 반환합니다. 보통 소켓 프로그램을 작성하는 거대한 과정의 첫 호출이 이것이며, 반환값은 이후 listen(), bind(), accept() 같은 여러 함수 호출에 사용할 수 있습니다.

일반적으로는 아래 예제처럼 getaddrinfo() 호출 결과에서 이 파라미터 값들을 얻습니다. 물론 정말 원한다면 손으로 직접 채워 넣어도 됩니다.

파라미터 설명
domain domain은 어떤 종류의 소켓이 필요한지를 나타냅니다. 믿기 어렵겠지만 정말 여러 값이 가능하지만, 이건 소켓 가이드이므로 보통 IPv4는 PF_INET, IPv6는 PF_INET6를 씁니다.
type type 파라미터도 여러 값이 가능하지만, 아마 대부분은 신뢰성 있는 TCP 소켓(send(), recv())용 SOCK_STREAM이나, 빠르지만 비신뢰적인 UDP 소켓(sendto(), recvfrom())용 SOCK_DGRAM을 쓰게 될 것입니다. (흥미로운 타입으로 SOCK_RAW도 있는데, 패킷을 직접 구성할 때 사용할 수 있습니다. 꽤 멋집니다.)
protocol 마지막으로 protocol 파라미터는 특정 소켓 타입과 함께 어떤 프로토콜을 쓸지 지정합니다. 이미 말했듯 예를 들어 SOCK_STREAM은 TCP를 사용합니다. 다행히 SOCK_STREAM이나 SOCK_DGRAM을 쓸 때는 프로토콜을 0으로 두기만 해도 알맞은 프로토콜이 자동으로 선택됩니다. 그렇지 않다면 getprotobyname()으로 적절한 프로토콜 번호를 찾을 수 있습니다.

반환값

이후 호출에서 사용할 새 소켓 디스크립터를 반환합니다. 오류가 발생하면 -1을 반환하며, errno가 알맞게 설정됩니다.

예제

struct addrinfo hints, *res;
int sockfd;

// first, load up address structs with getaddrinfo():

memset(&hints, 0, sizeof hints);
hints.ai_family = AF_UNSPEC;     // AF_INET, AF_INET6, or AF_UNSPEC
hints.ai_socktype = SOCK_STREAM; // SOCK_STREAM or SOCK_DGRAM

getaddrinfo("www.example.com", "3490", &hints, &res);

// make a socket using the information gleaned from getaddrinfo():
sockfd = socket(res->ai_family, res->ai_socktype, res->ai_protocol);

9.24 struct sockaddr와 친구들

인터넷 주소를 다루기 위한 구조체들

형식

#include <netinet/in.h>

// All pointers to socket address structures are often cast to
// pointers to this type before use in various functions and system
// calls:

struct sockaddr {
    unsigned short    sa_family;    // address family, AF_xxx
    char              sa_data[14];  // 14 bytes of protocol address
};


// IPv4 AF_INET sockets:

struct sockaddr_in {
    short            sin_family;   // e.g. AF_INET, AF_INET6
    unsigned short   sin_port;     // e.g. htons(3490)
    struct in_addr   sin_addr;     // see struct in_addr, below
    char             sin_zero[8];  // zero this if you want to
};

struct in_addr {
    unsigned long s_addr;          // load with inet_pton()
};


// IPv6 AF_INET6 sockets:

struct sockaddr_in6 {
    u_int16_t       sin6_family;   // address family, AF_INET6
    u_int16_t       sin6_port;     // port number, network order
    u_int32_t       sin6_flowinfo; // IPv6 flow information
    struct in6_addr sin6_addr;     // IPv6 address
    u_int32_t       sin6_scope_id; // Scope ID
};

struct in6_addr {
    unsigned char   s6_addr[16];   // load with inet_pton()
};


// General socket address holding structure, big enough to hold
// either struct sockaddr_in or struct sockaddr_in6 data:

struct sockaddr_storage {
    sa_family_t  ss_family;     // address family

    // all this is padding, implementation specific, ignore it:
    char      __ss_pad1[_SS_PAD1SIZE];
    int64_t   __ss_align;
    char      __ss_pad2[_SS_PAD2SIZE];
};

설명

이 구조체들은 인터넷 주소를 다루는 모든 시스템 호출과 함수의 기본 구조입니다. 대개는 getaddrinfo()가 이 구조체들을 채워 주고, 여러분은 필요할 때 그 안의 내용을 읽게 됩니다.

메모리 안에서 struct sockaddr_in과 struct sockaddr_in6은 struct sockaddr와 같은 형태의 시작 부분을 공유하므로, 우주의 종말만 아니라면 한 타입의 포인터를 다른 타입으로 자유롭게 캐스팅해도 해가 없습니다.

물론 우주 종말 이야기는 농담입니다. 만약 struct sockaddr_in*를 struct sockaddr*로 캐스팅하는 순간 우주가 끝난다면, 그건 순전히 우연일 테니 너무 걱정하지 마세요.

그러니 함수가 struct sockaddr*를 받는다고 하면, 여러분의 struct sockaddr_in*, struct sockaddr_in6*, 또는 struct sockaddr_storage*를 그 타입으로 쉽고 안전하게 캐스팅할 수 있다는 점을 기억하세요.

struct sockaddr_in은 IPv4 주소(예: “192.0.2.10”)에 사용하는 구조체입니다. 주소 계열(AF_INET), sin_port에 들어가는 포트, 그리고 sin_addr에 들어가는 IPv4 주소를 담습니다.

struct sockaddr_in에는 sin_zero 필드도 있는데, 어떤 사람들은 이 값을 0으로 설정해야 한다고 주장합니다. 다른 사람들은 아무 말도 하지 않고(Linux 문서에는 아예 언급도 없습니다), 실제로 꼭 0으로 채워야 하는 것 같지도 않습니다. 그래도 그러고 싶다면 memset()으로 0으로 채우세요.

struct in_addr는 시스템마다 꽤 괴상한 물건일 수 있습니다. 온갖 #define과 기타 수상한 것들이 붙은 기묘한 union인 경우도 있습니다. 하지만 여러분은 이 구조체에서 s_addr 필드만 사용하면 됩니다. 많은 시스템이 그 필드만 구현하기 때문입니다.

struct sockaddr_in6과 struct in6_addr도 매우 비슷하지만, 이쪽은 IPv6에 사용됩니다.

struct sockaddr_storage는 IP 버전에 독립적인 코드를 작성할 때, 새 주소가 IPv4일지 IPv6일지 모르는 상황에서 accept()나 recvfrom()에 넘길 수 있는 구조체입니다. 원래의 작은 struct sockaddr와 달리, struct sockaddr_storage는 두 타입을 모두 담을 만큼 충분히 큽니다.

예제

// IPv4:

struct sockaddr_in ip4addr;
int s;

ip4addr.sin_family = AF_INET;
ip4addr.sin_port = htons(3490);
inet_pton(AF_INET, "10.0.0.1", &ip4addr.sin_addr);

s = socket(PF_INET, SOCK_STREAM, 0);
bind(s, (struct sockaddr*)&ip4addr, sizeof ip4addr);
// IPv6:

struct sockaddr_in6 ip6addr;
int s;

ip6addr.sin6_family = AF_INET6;
ip6addr.sin6_port = htons(4950);
inet_pton(AF_INET6, "2001:db8:8714:3a90::12", &ip6addr.sin6_addr);

s = socket(PF_INET6, SOCK_STREAM, 0);
bind(s, (struct sockaddr*)&ip6addr, sizeof ip6addr);
