그럼 이제 소켓과 관련해 조금 더 비범한 주제들로 들어가 봅시다. 마음껏 파고들어 보세요!

7.1 블로킹

블로킹. 이름은 들어봤겠지만, 대체 그게 뭘까요? 한마디로 말하면 “block”은 기술자들이 “잠든다”는 뜻으로 쓰는 말입니다. 앞의 listener를 실행해 보면 패킷이 도착할 때까지 그냥 가만히 있는 것을 보셨을 겁니다. 실제로는 recvfrom()를 호출했는데 읽을 데이터가 없어서, 데이터가 도착할 때까지 그 자리에서 “블록된다”, 즉 잠든 상태가 되는 것입니다.

블록되는 함수는 많습니다. accept()도 블록되고, 모든 recv() 계열 함수도 블록됩니다. 이렇게 동작하는 이유는 원래 그렇게 허용되어 있기 때문입니다. 처음 socket()으로 소켓 디스크립터를 만들면 커널이 기본적으로 그것을 블로킹 모드로 설정합니다. 소켓이 블로킹되지 않게 하려면 fcntl()을 호출해야 합니다.

소켓을 논블로킹으로 설정하면 사실상 소켓에 정보가 있는지 계속 “폴링”할 수 있습니다. 논블로킹 소켓에서 읽기를 시도했는데 데이터가 없다면, 그 호출은 블록될 수 없으므로 -1을 반환하고 errno는 EAGAIN 또는 EWOULDBLOCK으로 설정됩니다.

(잠깐, EAGAIN이 올 수도 있고 EWOULDBLOCK이 올 수도 있다고요? 그럼 뭘 확인해야 하죠? 명세는 시스템이 어느 쪽을 반환할지 정확히 보장하지 않으므로, 이식성을 생각한다면 둘 다 확인해야 합니다.)

하지만 일반적으로 말해 이런 식의 폴링은 좋은 생각이 아닙니다. 소켓에 데이터가 있는지 보려고 프로그램을 바쁜 대기 상태에 넣어 두면 CPU 시간을 엄청나게 잡아먹게 됩니다. 읽을 데이터가 대기 중인지 우아하게 확인하는 더 나은 방법은 다음 절에서 설명할 poll()입니다.

7.2 poll()—동기식 I/O 멀티플렉싱

우리가 정말 하고 싶은 일은 소켓을 한 무더기 한꺼번에 감시한 다음, 실제로 데이터가 준비된 소켓만 처리하는 것입니다. 이렇게 하면 어떤 소켓이 읽을 준비가 되었는지 보려고 모든 소켓을 끊임없이 검사할 필요가 없습니다.

주의 한마디: poll()은 연결 수가 엄청 많아지면 끔찍할 정도로 느려집니다. 그런 상황에서는 시스템에서 가능한 가장 빠른 방법을 사용하려고 하는 libevent28 같은 이벤트 라이브러리를 쓰는 편이 더 좋은 성능을 냅니다.

그렇다면 폴링을 어떻게 피할 수 있을까요? 약간의 말장난 같지만, poll() 시스템 호출을 사용하면 됩니다. 요지는 운영체제에게 지저분한 일을 맡기고, 어떤 소켓에 읽을 데이터가 준비되었는지만 알려 달라고 하는 것입니다. 그동안 우리 프로세스는 잠들어 있을 수 있으니 시스템 자원도 절약됩니다.

전체적인 전략은 어떤 소켓 디스크립터를 감시할지, 어떤 이벤트를 기다릴지에 대한 정보를 담은 struct pollfd 배열을 유지하는 것입니다. 운영체제는 그중 하나의 이벤트가 발생할 때까지(예: “소켓 읽기 가능!”) 또는 사용자가 지정한 타임아웃이 될 때까지 poll() 호출에서 대기합니다.

유용한 점은, listen() 중인 소켓은 새로운 연결이 들어와 accept()할 준비가 되었을 때 “읽을 준비 완료” 상태를 반환한다는 것입니다.

말은 이쯤 하고, 실제로 어떻게 쓰는지 봅시다.

#include <poll.h>

int poll(struct pollfd fds[], nfds_t nfds, int timeout);

fds는 어떤 소켓을 무엇 때문에 감시할지에 대한 정보 배열이고, nfds는 그 배열의 원소 수, timeout은 밀리초 단위 타임아웃입니다. 반환값은 이벤트가 발생한 배열 원소의 개수입니다.

그 struct를 한 번 봅시다.

struct pollfd {
    int fd;         // the socket descriptor
    short events;   // bitmap of events we're interested in
    short revents;  // on return, bitmap of events that occurred
};

이 구조체들의 배열을 하나 만들고, 각 원소의 fd 필드에 우리가 감시하고 싶은 소켓 디스크립터를 넣습니다. 그리고 events 필드에 관심 있는 이벤트 종류를 지정합니다.

events 필드는 다음 값들의 비트 OR입니다.

매크로 설명
POLLIN 이 소켓에서 recv()할 데이터가 준비되면 알려 줍니다.
POLLOUT 이 소켓에 블로킹 없이 send()할 수 있게 되면 알려 줍니다.
POLLHUP 원격 측이 연결을 닫았을 때 알려 줍니다.

struct pollfd 배열을 준비했다면 배열 크기와 밀리초 단위 타임아웃 값과 함께 그것을 poll()에 넘기면 됩니다. (음수 타임아웃을 주면 무한히 기다릴 수 있습니다.)

poll()이 반환된 뒤에는 revents 필드를 확인해서 POLLIN이나 POLLOUT가 설정되었는지 보고 어떤 이벤트가 일어났는지 알 수 있습니다.

(실제로는 poll()로 할 수 있는 일이 더 많습니다. 자세한 내용은 아래의 poll() 매뉴얼 페이지를 참고하세요.)

다음은 예제29로, 표준 입력에서 읽을 데이터가 준비될 때까지, 즉 RETURN 키를 누를 때까지 2.5초 기다립니다.

#include <stdio.h>
#include <poll.h>

int main(void)
{
    struct pollfd pfds[1]; // More if you want to monitor more

    pfds[0].fd = 0;          // Standard input
    pfds[0].events = POLLIN; // Tell me when ready to read

    // If you needed to monitor other things, as well:
    //pfds[1].fd = some_socket; // Some socket descriptor
    //pfds[1].events = POLLIN;  // Tell me when ready to read

    printf("Hit RETURN or wait 2.5 seconds for timeout\n");

    int num_events = poll(pfds, 1, 2500); // 2.5 second timeout

    if (num_events == 0) {
        printf("Poll timed out!\n");
    } else {
        int pollin_happened = pfds[0].revents & POLLIN;

        if (pollin_happened) {
            printf("File descriptor %d is ready to read\n",
                    pfds[0].fd);
        } else {
            printf("Unexpected event occurred: %d\n",
                    pfds[0].revents);
        }
    }

    return 0;
}

다시 한 번 보세요. poll()은 이벤트가 발생한 pfds 배열 원소의 개수를 반환합니다. 배열의 어느 원소인지까지 알려 주는 것은 아니므로 직접 훑어봐야 하지만, 최소한 revents가 0이 아닌 항목이 몇 개인지는 알려 주므로 그만큼 찾으면 스캔을 멈출 수 있습니다.

여기서 몇 가지 의문이 생길 수 있습니다. poll()에 넘길 집합에 새 파일 디스크립터를 어떻게 추가할까요? 필요한 만큼 배열 공간을 충분히 확보해 두거나, 필요할 때 realloc()으로 늘리면 됩니다.

그렇다면 항목 삭제는요? 삭제할 위치 위에 배열의 마지막 원소를 덮어쓰고, poll()에 넘기는 개수를 하나 줄이면 됩니다. 또는 어떤 fd 필드를 음수로 설정해 두면 poll()이 그것을 무시합니다.

이걸 전부 합쳐서 telnet으로 접속 가능한 채팅 서버를 만들려면 어떻게 해야 할까요?

우선 리스너 소켓을 하나 만들고 그것을 poll()할 파일 디스크립터 집합에 넣습니다. (들어오는 연결이 있으면 읽을 준비가 된 것으로 표시됩니다.)

그다음 새 연결이 생길 때마다 struct pollfd 배열에 추가합니다. 공간이 부족하면 동적으로 배열을 늘립니다.

연결이 닫히면 배열에서 제거합니다.

그리고 어떤 연결이 읽을 준비가 되면 데이터를 읽어, 다른 모든 연결에 보내서 다른 사용자가 입력한 내용을 볼 수 있게 합니다.

그러니 이 poll 서버30를 한번 실행해 보세요. 한 창에서 서버를 띄우고, 다른 여러 터미널 창에서 telnet localhost 9034로 접속하면 됩니다. 한 창에 입력한 내용이 다른 창들에도 보여야 합니다. (RETURN을 누른 뒤에요.)

그뿐만 아니라, CTRL-]를 누르고 quit을 입력해 telnet을 종료하면 서버가 연결 종료를 감지해서 파일 디스크립터 배열에서 해당 항목을 제거해야 합니다.

/*
** pollserver.c -- a cheezy multiperson chat server
*/

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>
#include <netdb.h>
#include <poll.h>

#define PORT "9034"   // Port we're listening on

/*
 * Convert socket to IP address string.
 * addr: struct sockaddr_in or struct sockaddr_in6
 */
const char *inet_ntop2(void *addr, char *buf, size_t size)
{
    struct sockaddr_storage *sas = addr;
    struct sockaddr_in *sa4;
    struct sockaddr_in6 *sa6;
    void *src;

    switch (sas->ss_family) {
        case AF_INET:
            sa4 = addr;
            src = &(sa4->sin_addr);
            break;
        case AF_INET6:
            sa6 = addr;
            src = &(sa6->sin6_addr);
            break;
        default:
            return NULL;
    }

    return inet_ntop(sas->ss_family, src, buf, size);
}

/*
 * Return a listening socket.
 */
int get_listener_socket(void)
{
    int listener;     // Listening socket descriptor
    int yes=1;        // For setsockopt() SO_REUSEADDR, below
    int rv;

    struct addrinfo hints, *ai, *p;

    // Get us a socket and bind it
    memset(&hints, 0, sizeof hints);
    hints.ai_family = AF_INET;
    hints.ai_socktype = SOCK_STREAM;
    hints.ai_flags = AI_PASSIVE;
    if ((rv = getaddrinfo(NULL, PORT, &hints, &ai)) != 0) {
        fprintf(stderr, "pollserver: %s\n", gai_strerror(rv));
        exit(1);
    }

    for(p = ai; p != NULL; p = p->ai_next) {
        listener = socket(p->ai_family, p->ai_socktype,
                p->ai_protocol);
        if (listener < 0) {
            continue;
        }

        // Lose the pesky "address already in use" error message
        setsockopt(listener, SOL_SOCKET, SO_REUSEADDR, &yes,
                sizeof(int));

        if (bind(listener, p->ai_addr, p->ai_addrlen) < 0) {
            close(listener);
            continue;
        }

        break;
    }

    // If we got here, it means we didn't get bound
    if (p == NULL) {
        return -1;
    }

    freeaddrinfo(ai); // All done with this

    // Listen
    if (listen(listener, 10) == -1) {
        return -1;
    }

    return listener;
}

/*
 * Add a new file descriptor to the set.
 */
void add_to_pfds(struct pollfd **pfds, int newfd, int *fd_count,
        int *fd_size)
{
    // If we don't have room, add more space in the pfds array
    if (*fd_count == *fd_size) {
        *fd_size *= 2; // Double it
        *pfds = realloc(*pfds, sizeof(**pfds) * (*fd_size));
    }

    (*pfds)[*fd_count].fd = newfd;
    (*pfds)[*fd_count].events = POLLIN; // Check ready-to-read
    (*pfds)[*fd_count].revents = 0;

    (*fd_count)++;
}

/*
 * Remove a file descriptor at a given index from the set.
 */
void del_from_pfds(struct pollfd pfds[], int i, int *fd_count)
{
    // Copy the one from the end over this one
    pfds[i] = pfds[*fd_count-1];

    (*fd_count)--;
}

/*
 * Handle incoming connections.
 */
void handle_new_connection(int listener, int *fd_count,
        int *fd_size, struct pollfd **pfds)
{
    struct sockaddr_storage remoteaddr; // Client address
    socklen_t addrlen;
    int newfd;  // Newly accept()ed socket descriptor
    char remoteIP[INET6_ADDRSTRLEN];

    addrlen = sizeof remoteaddr;
    newfd = accept(listener, (struct sockaddr *)&remoteaddr,
            &addrlen);

    if (newfd == -1) {
        perror("accept");
    } else {
        add_to_pfds(pfds, newfd, fd_count, fd_size);

        printf("pollserver: new connection from %s on socket %d\n",
                inet_ntop2(&remoteaddr, remoteIP, sizeof remoteIP),
                newfd);
    }
}

/*
 * Handle regular client data or client hangups.
 */
void handle_client_data(int listener, int *fd_count,
        struct pollfd *pfds, int *pfd_i)
{
    char buf[256];    // Buffer for client data

    int nbytes = recv(pfds[*pfd_i].fd, buf, sizeof buf, 0);

    int sender_fd = pfds[*pfd_i].fd;

    if (nbytes <= 0) { // Got error or connection closed by client
        if (nbytes == 0) {
            // Connection closed
            printf("pollserver: socket %d hung up\n", sender_fd);
        } else {
            perror("recv");
        }

        close(pfds[*pfd_i].fd); // Bye!

        del_from_pfds(pfds, *pfd_i, fd_count);

        // reexamine the slot we just deleted
        (*pfd_i)--;

    } else { // We got some good data from a client
        printf("pollserver: recv from fd %d: %.*s", sender_fd,
                nbytes, buf);
        // Send to everyone!
        for(int j = 0; j < *fd_count; j++) {
            int dest_fd = pfds[j].fd;

            // Except the listener and ourselves
            if (dest_fd != listener && dest_fd != sender_fd) {
                if (send(dest_fd, buf, nbytes, 0) == -1) {
                    perror("send");
                }
            }
        }
    }
}

/*
 * Process all existing connections.
 */
void process_connections(int listener, int *fd_count, int *fd_size,
        struct pollfd **pfds)
{
    for(int i = 0; i < *fd_count; i++) {

        // Check if someone's ready to read
        if ((*pfds)[i].revents & (POLLIN | POLLHUP)) {
            // We got one!!

            if ((*pfds)[i].fd == listener) {
                // If we're the listener, it's a new connection
                handle_new_connection(listener, fd_count, fd_size,
                        pfds);
            } else {
                // Otherwise we're just a regular client
                handle_client_data(listener, fd_count, *pfds, &i);
            }
        }
    }
}

/*
 * Main: create a listener and connection set, loop forever
 * processing connections.
 */
int main(void)
{
    int listener;     // Listening socket descriptor

    // Start off with room for 5 connections
    // (We'll realloc as necessary)
    int fd_size = 5;
    int fd_count = 0;
    struct pollfd *pfds = malloc(sizeof *pfds * fd_size);

    // Set up and get a listening socket
    listener = get_listener_socket();

    if (listener == -1) {
        fprintf(stderr, "error getting listening socket\n");
        exit(1);
    }

    // Add the listener to set;
    // Report ready to read on incoming connection
    pfds[0].fd = listener;
    pfds[0].events = POLLIN;

    fd_count = 1; // For the listener

    puts("pollserver: waiting for connections...");

    // Main loop
    for(;;) {
        int poll_count = poll(pfds, fd_count, -1);

        if (poll_count == -1) {
            perror("poll");
            exit(1);
        }

        // Run through connections looking for data to read
        process_connections(listener, &fd_count, &fd_size, &pfds);
    }

    free(pfds);
}

다음 절에서는 비슷하지만 더 오래된 함수인 select()를 살펴보겠습니다. select()와 poll()은 기능과 성능이 비슷하며, 실제 차이는 주로 사용 방식에 있습니다. select()가 약간 더 이식성이 좋을 수는 있지만, 사용감은 조금 더 투박할 수 있습니다. 여러분 시스템에서 지원하기만 한다면 더 마음에 드는 쪽을 선택하면 됩니다.

7.3 select()—동기식 I/O 멀티플렉싱, 고전 방식

이 함수는 다소 이상하게 보일 수 있지만 아주 유용합니다. 다음 상황을 생각해 봅시다. 여러분은 서버이고, 들어오는 새 연결을 듣는 동시에 이미 연결된 소켓들에서도 계속 읽어야 합니다.

“문제없죠. accept() 한 번 하고 recv() 몇 번 하면 되잖아요.”라고 생각할 수도 있습니다. 하지만 그렇게 단순하지 않습니다. accept()에서 블록된 상태라면 동시에 어떻게 recv()를 하죠? “논블로킹 소켓을 쓰면 되잖아요!” 하지만 그렇게 하면 CPU를 낭비하고 싶지 않겠죠. 그럼 어떻게 해야 할까요?

select()는 여러 소켓을 동시에 감시할 수 있게 해 줍니다. 어떤 소켓은 읽기 가능, 어떤 소켓은 쓰기 가능, 그리고 원한다면 어떤 소켓에 예외가 발생했는지도 알려 줍니다.

주의 한마디: select()는 이식성은 좋지만, 연결 수가 엄청 많아지면 매우 느려집니다. 그런 경우에는 시스템에서 가능한 가장 빠른 방법을 선택하려는 libevent31 같은 이벤트 라이브러리를 사용하는 편이 더 낫습니다.

더 길게 끌지 말고, 바로 select() 원형을 보겠습니다.

#include <sys/time.h>
#include <sys/types.h>
#include <unistd.h>

int select(int numfds, fd_set *readfds, fd_set *writefds,
           fd_set *exceptfds, struct timeval *timeout);

이 함수는 파일 디스크립터의 “집합”을 감시합니다. 구체적으로는 readfds, writefds, exceptfds입니다. 표준 입력과 어떤 소켓 디스크립터 sockfd에서 읽을 수 있는지 알고 싶다면, 파일 디스크립터 0과 sockfd를 readfds 집합에 넣으면 됩니다. numfds 매개변수는 가장 큰 파일 디스크립터 값에 1을 더한 값으로 설정해야 합니다. 이 예에서는 sockfd가 표준 입력(0)보다 확실히 크므로 sockfd+1이 됩니다.

select()가 반환되면 readfds는 여러분이 고른 디스크립터 중 어떤 것이 읽기 가능한지를 반영하도록 수정됩니다. 이것은 아래의 FD_ISSET() 매크로로 검사할 수 있습니다.

더 나아가기 전에 이 집합들을 어떻게 다루는지부터 보겠습니다. 각 집합의 타입은 fd_set이고, 다음 매크로들이 여기에 사용됩니다.

함수 설명
FD_SET(int fd, fd_set *set); fd를 set에 추가합니다.
FD_CLR(int fd, fd_set *set); fd를 set에서 제거합니다.
FD_ISSET(int fd, fd_set *set); fd가 set 안에 있으면 참을 반환합니다.
FD_ZERO(fd_set *set); set의 모든 항목을 지웁니다.

마지막으로 이 괴상하게 생긴 struct timeval은 뭘까요? 때로는 누군가가 데이터를 보내 줄 때까지 무한히 기다리고 싶지 않을 수 있습니다. 예를 들어 아무 일도 없어도 96초마다 터미널에 “Still Going…” 같은 메시지를 출력하고 싶을 수도 있죠. 이 시간 구조체를 사용하면 타임아웃을 지정할 수 있습니다. 시간이 초과됐는데도 select()가 준비된 파일 디스크립터를 찾지 못하면 반환해서 나머지 처리를 계속할 수 있게 해 줍니다.

struct timeval에는 다음 필드가 있습니다.

struct timeval {
    int tv_sec;     // seconds
    int tv_usec;    // microseconds
};

tv_sec에는 기다릴 초 수를, tv_usec에는 기다릴 마이크로초 수를 넣으면 됩니다. 네, 밀리초가 아니라 _마이크로_초입니다. 1밀리초는 1,000마이크로초이고, 1초는 1,000밀리초이므로 1초는 1,000,000마이크로초입니다. 왜 이름이 “usec”일까요? 여기서 “u”는 “micro”를 뜻하는 그리스 문자 μ(Mu)를 흉내 낸 것입니다. 그리고 함수가 반환될 때 timeout이 남은 시간을 보여 주도록 갱신될 수도 있습니다. 이 부분은 어떤 Unix 계열을 쓰느냐에 따라 다릅니다.

야호, 마이크로초 해상도의 타이머라니! 라고 하고 싶지만 너무 기대하진 마세요. struct timeval을 아무리 작게 설정해도 표준 Unix 타임슬라이스의 일부만큼은 기다리게 될 가능성이 큽니다.

그 외 알아둘 점도 있습니다. struct timeval의 필드를 모두 0으로 두면 select()는 즉시 타임아웃되어 사실상 집합의 모든 파일 디스크립터를 폴링합니다. timeout 인자를 NULL로 두면 타임아웃 없이 첫 번째 파일 디스크립터가 준비될 때까지 기다립니다. 마지막으로, 어떤 집합은 기다릴 필요가 없다면 select() 호출에서 그 인자를 그냥 NULL로 두면 됩니다.

다음 코드 조각32은 표준 입력에 무언가 나타날 때까지 2.5초 기다립니다.

/*
** select.c -- a select() demo
*/

#include <stdio.h>
#include <sys/time.h>
#include <sys/types.h>
#include <unistd.h>

#define STDIN 0  // file descriptor for standard input

int main(void)
{
    struct timeval tv;
    fd_set readfds;

    tv.tv_sec = 2;
    tv.tv_usec = 500000;

    FD_ZERO(&readfds);
    FD_SET(STDIN, &readfds);

    // don't care about writefds and exceptfds:
    select(STDIN+1, &readfds, NULL, NULL, &tv);

    if (FD_ISSET(STDIN, &readfds))
        printf("A key was pressed!\n");
    else
        printf("Timed out.\n");

    return 0;
}

라인 버퍼링 터미널을 쓰고 있다면 RETURN 키를 눌러야 하고, 그렇지 않으면 결국 타임아웃될 수 있습니다.

이걸 보고 데이터그램 소켓에서 데이터를 기다리기에 딱 좋은 방법이라고 생각할 수도 있습니다. 실제로 그럴 수도 있습니다. 어떤 Unix에서는 이런 식으로 select를 사용할 수 있지만, 어떤 Unix에서는 안 됩니다. 시도해 보고 싶다면 로컬 매뉴얼 페이지를 확인해 보세요.

일부 Unix는 struct timeval 안의 시간을 남은 타임아웃 시간으로 갱신해 주지만, 다른 Unix는 그렇지 않습니다. 이식성을 원한다면 그런 동작에 의존하지 마세요. (경과 시간을 추적해야 한다면 gettimeofday()를 사용하세요. 번거롭지만 현실이 그렇습니다.)

읽기 집합 안에 있는 소켓이 연결을 닫으면 어떻게 될까요? 그 경우 select()는 그 소켓 디스크립터를 “읽을 준비 완료” 상태로 표시한 채 반환합니다. 그리고 실제로 recv()를 해 보면 recv()가 0을 반환합니다. 이걸로 클라이언트가 연결을 닫았다는 것을 알 수 있습니다.

select()에 대해 하나 더 흥미로운 점이 있습니다. listen() 중인 소켓이 있다면, 그 소켓의 파일 디스크립터를 readfds 집합에 넣어 새 연결이 들어왔는지 확인할 수 있습니다.

이것으로 강력한 select() 함수에 대한 빠른 개요는 끝입니다.

하지만 요청이 많았으니 이번에는 좀 더 깊이 있는 예제를 보겠습니다. 안타깝게도 위의 아주 단순한 예제와 지금 볼 예제 사이의 차이는 꽤 큽니다. 그래도 먼저 코드를 보고, 이어지는 설명을 읽어 보세요.

이 프로그램33은 단순한 다중 사용자 채팅 서버처럼 동작합니다. 한 창에서 서버를 실행한 뒤 다른 여러 창에서 telnet으로 (“telnet hostname 9034”) 접속하세요. 한 telnet 세션에서 입력한 내용이 다른 모든 세션에도 보여야 합니다.

/*
** selectserver.c -- a cheezy multiperson chat server
*/

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>
#include <netdb.h>

#define PORT "9034"   // port we're listening on

/*
 * Convert socket to IP address string.
 * addr: struct sockaddr_in or struct sockaddr_in6
 */
const char *inet_ntop2(void *addr, char *buf, size_t size)
{
    struct sockaddr_storage *sas = addr;
    struct sockaddr_in *sa4;
    struct sockaddr_in6 *sa6;
    void *src;

    switch (sas->ss_family) {
        case AF_INET:
            sa4 = addr;
            src = &(sa4->sin_addr);
            break;
        case AF_INET6:
            sa6 = addr;
            src = &(sa6->sin6_addr);
            break;
        default:
            return NULL;
    }

    return inet_ntop(sas->ss_family, src, buf, size);
}

/*
 * Return a listening socket
 */
int get_listener_socket(void)
{
    struct addrinfo hints, *ai, *p;
    int yes=1;    // for setsockopt() SO_REUSEADDR, below
    int rv;
    int listener;

    // get us a socket and bind it
    memset(&hints, 0, sizeof hints);
    hints.ai_family = AF_UNSPEC;
    hints.ai_socktype = SOCK_STREAM;
    hints.ai_flags = AI_PASSIVE;
    if ((rv = getaddrinfo(NULL, PORT, &hints, &ai)) != 0) {
        fprintf(stderr, "selectserver: %s\n", gai_strerror(rv));
        exit(1);
    }

    for(p = ai; p != NULL; p = p->ai_next) {
        listener = socket(p->ai_family, p->ai_socktype,
                p->ai_protocol);
        if (listener < 0) {
            continue;
        }

        // lose the pesky "address already in use" error message
        setsockopt(listener, SOL_SOCKET, SO_REUSEADDR, &yes,
                sizeof(int));

        if (bind(listener, p->ai_addr, p->ai_addrlen) < 0) {
            close(listener);
            continue;
        }

        break;
    }

    // if we got here, it means we didn't get bound
    if (p == NULL) {
        fprintf(stderr, "selectserver: failed to bind\n");
        exit(2);
    }

    freeaddrinfo(ai); // all done with this

    // listen
    if (listen(listener, 10) == -1) {
        perror("listen");
        exit(3);
    }

    return listener;
}

/*
 * Add new incoming connections to the proper sets
 */
void handle_new_connection(int listener, fd_set *master, int *fdmax)
{
    socklen_t addrlen;
    int newfd;        // newly accept()ed socket descriptor
    struct sockaddr_storage remoteaddr; // client address
    char remoteIP[INET6_ADDRSTRLEN];

    addrlen = sizeof remoteaddr;
    newfd = accept(listener,
        (struct sockaddr *)&remoteaddr,
        &addrlen);

    if (newfd == -1) {
        perror("accept");
    } else {
        FD_SET(newfd, master); // add to master set
        if (newfd > *fdmax) {  // keep track of the max
            *fdmax = newfd;
        }
        printf("selectserver: new connection from %s on "
            "socket %d\n",
            inet_ntop2(&remoteaddr, remoteIP, sizeof remoteIP),
            newfd);
    }
}

/*
 * Broadcast a message to all clients
 */
void broadcast(char *buf, int nbytes, int listener, int s,
               fd_set *master, int fdmax)
{
    for(int j = 0; j <= fdmax; j++) {
        // send to everyone!
        if (FD_ISSET(j, master)) {
            // except the listener and ourselves
            if (j != listener && j != s) {
                if (send(j, buf, nbytes, 0) == -1) {
                    perror("send");
                }
            }
        }
    }
}

/*
 * Handle client data and hangups
 */
void handle_client_data(int s, int listener, fd_set *master,
                        int fdmax)
{
    char buf[256];    // buffer for client data
    int nbytes;

    // handle data from a client
    if ((nbytes = recv(s, buf, sizeof buf, 0)) <= 0) {
        // got error or connection closed by client
        if (nbytes == 0) {
            // connection closed
            printf("selectserver: socket %d hung up\n", s);
        } else {
            perror("recv");
        }
        close(s); // bye!
        FD_CLR(s, master); // remove from master set
    } else {
        // we got some data from a client
        broadcast(buf, nbytes, listener, s, master, fdmax);
    }
}

/*
 * Main
 */
int main(void)
{
    fd_set master;    // master file descriptor list
    fd_set read_fds;  // temp file descriptor list for select()
    int fdmax;        // maximum file descriptor number

    int listener;     // listening socket descriptor

    FD_ZERO(&master);    // clear the master and temp sets
    FD_ZERO(&read_fds);

    listener = get_listener_socket();

    // add the listener to the master set
    FD_SET(listener, &master);

    // keep track of the biggest file descriptor
    fdmax = listener; // so far, it's this one

    // main loop
    for(;;) {
        read_fds = master; // copy it
        if (select(fdmax+1, &read_fds, NULL, NULL, NULL) == -1) {
            perror("select");
            exit(4);
        }

        // run through the existing connections looking for data
        // to read
        for(int i = 0; i <= fdmax; i++) {
            if (FD_ISSET(i, &read_fds)) { // we got one!!
                if (i == listener)
                    handle_new_connection(i, &master, &fdmax);
                else
                    handle_client_data(i, listener, &master, fdmax);
            }
        }
    }

    return 0;
}

코드를 보면 파일 디스크립터 집합이 두 개 있습니다. master와 read_fds입니다. 첫 번째인 master에는 현재 연결된 모든 소켓 디스크립터와 새 연결을 기다리는 리스너 소켓 디스크립터가 들어 있습니다.

master 집합을 따로 두는 이유는 select()가 넘겨받은 집합을 실제로 변경해서 어떤 소켓이 읽기 준비가 되었는지를 반영하기 때문입니다. 저는 한 번의 select() 호출에서 다음 호출까지 연결 상태를 계속 추적해야 하므로, 원본 집합을 안전한 곳에 따로 보관해야 합니다. 그래서 마지막 순간에 master를 read_fds에 복사한 뒤 select()를 호출합니다.

그렇다면 새 연결이 들어올 때마다 master 집합에 추가해야 한다는 뜻인가요? 맞습니다. 연결이 닫힐 때마다 master에서 제거해야 하냐고요? 그것도 맞습니다.

제가 listener 소켓이 읽기 준비가 되었는지 확인하는 것을 보세요. 준비가 되었다면 새 연결이 대기 중이라는 뜻이고, 그것을 accept()해서 master 집합에 추가합니다. 마찬가지로 클라이언트 연결이 읽기 준비가 되었는데 recv()가 0을 반환하면, 클라이언트가 연결을 닫았다는 뜻이므로 master에서 제거해야 합니다.

반대로 클라이언트의 recv()가 0이 아닌 값을 반환하면 데이터가 도착했다는 뜻입니다. 그 데이터를 읽은 뒤 master 목록을 훑어 다른 모든 연결된 클라이언트에게 보내면 됩니다.

이것으로 강력한 select() 함수에 대한, 말 그대로 아주 간단하지만은 않은 개요가 끝났습니다.

Linux 팬들을 위한 짧은 메모도 하나 남기겠습니다. 드물긴 하지만 Linux의 select()는 “읽을 준비 완료”라고 반환해 놓고 실제로는 읽을 준비가 안 되어 있는 경우가 있습니다. 즉, select()는 안 막힌다고 했는데 뒤이은 read()가 블록될 수 있다는 뜻입니다. 어이없죠. 우회책은 수신 소켓에 O_NONBLOCK 플래그를 설정해서 EWOULDBLOCK 오류가 나게 하는 것입니다. 그 오류가 발생하면 안전하게 무시할 수 있습니다. 소켓을 논블로킹으로 설정하는 방법은 fcntl() 참고 페이지를 보세요.

덤으로 하나 더 말하자면, select()와 거의 비슷하게 동작하지만 파일 디스크립터 집합 관리 방식이 다른 poll()이라는 함수도 있습니다. 한번 살펴보세요!

7.4 부분적인 send() 처리

앞의 send() 절에서 send()가 요청한 바이트를 모두 보내지 못할 수 있다고 말했던 것 기억하시나요? 예를 들어 512바이트를 보내고 싶었는데 412를 반환하는 경우 말입니다. 그럼 나머지 100바이트는 어떻게 된 걸까요?

그 바이트들은 여전히 여러분의 버퍼 안에서 전송되기를 기다리고 있습니다. 여러분이 통제할 수 없는 여러 사정 때문에 커널이 데이터를 한 번에 다 보내지 않기로 결정한 것이고, 이제 남은 데이터를 끝까지 보내는 일은 여러분 몫입니다.

이를 위해 이런 함수를 만들 수 있습니다.

#include <sys/types.h>
#include <sys/socket.h>

int sendall(int s, char *buf, int *len)
{
    int total = 0;        // how many bytes we've sent
    int bytesleft = *len; // how many we have left to send
    int n;

    while(total < *len) {
        n = send(s, buf+total, bytesleft, 0);
        if (n == -1) { break; }
        total += n;
        bytesleft -= n;
    }

    *len = total; // return number actually sent here

    return n==-1?-1:0; // return -1 on failure, 0 on success
}

이 예제에서 s는 데이터를 보낼 소켓이고, buf는 데이터를 담고 있는 버퍼, len은 버퍼 안 바이트 수를 담은 int에 대한 포인터입니다.

이 함수는 오류가 나면 -1을 반환하고, errno는 send() 호출 당시의 값을 그대로 유지합니다. 실제로 보낸 바이트 수는 len에 반환됩니다. 오류만 없다면 이 값은 원래 보내려던 바이트 수와 같아야 합니다. sendall()은 어떻게든 데이터를 끝까지 보내려고 최선을 다하지만, 오류가 생기면 즉시 그 사실을 알려 줍니다.

완전성을 위해 함수 호출 예제도 하나 보겠습니다.

char buf[10] = "Beej!";
int len;

len = strlen(buf);
if (sendall(s, buf, &len) == -1) {
    perror("sendall");
    printf("We only sent %d bytes because of the error!\n", len);
}

그렇다면 수신 측에서는 패킷 일부만 도착했을 때 어떻게 될까요? 패킷 길이가 가변적이라면 수신자는 어디서 한 패킷이 끝나고 다음 패킷이 시작되는지 어떻게 알 수 있을까요? 그렇습니다. 현실의 네트워크는 꽤 성가십니다. 아마 캡슐화가 필요할 겁니다. (문서 초반의 데이터 캡슐화 절 기억하시나요?) 자세한 내용은 계속 읽어 보세요.

7.5 직렬화—데이터를 패킹하는 방법

지금까지 해 보니 텍스트 데이터를 네트워크로 보내는 것은 충분히 쉽다는 걸 알게 되었을 겁니다. 하지만 int나 float 같은 “바이너리” 데이터를 보내고 싶다면 어떻게 해야 할까요? 방법은 몇 가지 있습니다.

  1. sprintf() 같은 함수로 숫자를 텍스트로 바꾼 뒤 그 텍스트를 보냅니다. 수신 측은 strtol() 같은 함수로 다시 숫자로 파싱합니다.

  2. 데이터 포인터를 send()에 넘겨 원시 형태 그대로 보냅니다.

  3. 숫자를 이식 가능한 바이너리 형식으로 인코딩합니다. 수신 측은 그것을 디코딩합니다.

첫 번째 방법인 숫자를 텍스트로 인코딩해서 보내는 방식은, 선로를 통해 흐르는 데이터를 사람이 직접 출력하고 읽기 쉽다는 장점이 있습니다. 대역폭이 큰 문제가 아닌 상황에서는, 예를 들어 IRC(Internet Relay Chat)34처럼 사람이 읽을 수 있는 프로토콜이 아주 훌륭할 수 있습니다. 하지만 변환이 느리고, 결과가 거의 항상 원래 숫자보다 더 많은 공간을 차지한다는 단점이 있습니다.

두 번째 방법은 원시 데이터를 그대로 전달하는 것입니다. 꽤 쉽습니다. (하지만 위험합니다!) 그냥 보낼 데이터의 포인터를 꺼내 send에 넘기면 됩니다.

double d = 3490.15926535;

send(s, &d, sizeof d, 0);  /* DANGER--non-portable! */

수신 측에서는 이렇게 받습니다.

double d;

recv(s, &d, sizeof d, 0);  /* DANGER--non-portable! */

빠르고 단순하니 뭐가 문제일까요? 문제는 모든 아키텍처가 double(그리고 int도 마찬가지입니다)를 같은 비트 표현이나 같은 바이트 순서로 저장하지는 않는다는 것입니다. 이 코드는 명백히 이식성이 없습니다. (물론 여러분에게 이식성이 중요하지 않다면 아주 빠르고 편한 방법이 될 수도 있습니다.)

정수형을 패킹할 때는 htons() 계열 함수가 숫자를 네트워크 바이트 순서로 바꿔 이식성을 높여 준다는 사실을 이미 보았습니다. 그리고 그게 올바른 방식이기도 합니다. 하지만 안타깝게도 float용으로는 그런 함수가 없습니다. 그럼 희망은 없는 걸까요?

걱정 마세요! (조금이라도 걱정하셨나요? 아니라고요?) 우리가 할 수 있는 일이 있습니다. 데이터를 수신 측이 원격지에서 다시 풀어낼 수 있는 알려진 바이너리 형식으로 패킹하는 것입니다. “marshal”, “serialize” 같은 다른 이름으로도 부르죠.

“알려진 바이너리 형식”이 무슨 뜻이냐고요? 이미 htons() 예제를 봤죠? 그 함수는 숫자를 호스트 형식에서 네트워크 바이트 순서로 바꿉니다. (원한다면 “인코딩”한다고 생각해도 됩니다.) 수신 측은 이를 되돌리기 위해 ntohs()를 호출합니다.

하지만 방금 다른 비정수형에는 그런 함수가 없다고 하지 않았냐고요? 맞습니다. 그렇게 말했습니다. 그리고 C에는 이를 위한 표준 방식이 없기 때문에 조금 난감한 상황이 됩니다.

해야 할 일은 데이터를 알려진 형식으로 패킹해서 선로로 보내고, 수신 측에서 그것을 디코딩하는 것입니다. 예를 들어 float를 패킹하려면, 빠르게 만들었고 개선 여지가 많은 이런 코드35가 있습니다.

#include <stdint.h>

uint32_t htonf(float f)
{
    uint32_t p;
    uint32_t sign;

    if (f < 0) { sign = 1; f = -f; }
    else { sign = 0; }
        
    // whole part and sign
    p = ((((uint32_t)f)&0x7fff)<<16) | (sign<<31);

    // fraction
    p |= (uint32_t)(((f - (int)f) * 65536.0f))&0xffff;

    return p;
}

float ntohf(uint32_t p)
{
    float f = ((p>>16)&0x7fff); // whole part
    f += (p&0xffff) / 65536.0f; // fraction

    if (((p>>31)&0x1) == 0x1) { f = -f; } // sign bit set

    return f;
}

위 코드는 float를 32비트 숫자에 저장하는, 다소 순진한 구현입니다. 최상위 비트(31)는 부호를 저장하고(“1”이면 음수), 그다음 비트들(30-16)은 float의 정수 부분을 저장합니다. 마지막으로 남은 비트들(15-0)은 소수 부분을 저장합니다.

사용법은 꽤 단순합니다.

#include <stdio.h>

int main(void)
{
    float f = 3.1415926, f2;
    uint32_t netf;

    netf = htonf(f);  // convert to "network" form
    f2 = ntohf(netf); // convert back to test

    printf("Original: %f\n", f);        // 3.141593
    printf(" Network: 0x%08X\n", netf); // 0x0003243F
    printf("Unpacked: %f\n", f2);       // 3.141586

    return 0;
}

장점은 작고, 단순하고, 빠르다는 것입니다. 단점은 공간 활용이 비효율적이고 표현 범위가 매우 제한된다는 점입니다. 32767보다 큰 수를 넣어 보세요. 별로 좋아하지 않을 겁니다. 또한 위 예제에서 마지막 소수 몇 자리가 정확히 보존되지 않는 것도 볼 수 있습니다.

그렇다면 대신 무엇을 할 수 있을까요? 부동소수점 수를 저장하는 표준 형식은 IEEE-75436입니다. 대부분의 컴퓨터는 내부적으로 부동소수점 연산에 이 형식을 사용하므로, 그런 경우 엄밀히 말하면 변환이 필요 없을 수도 있습니다. 하지만 소스 코드를 이식 가능하게 만들고 싶다면 그렇게 가정할 수만은 없습니다. (반대로 속도가 중요하다면, 변환이 필요 없는 플랫폼에서는 그 비용을 최적화로 제거하는 것이 좋습니다. htons() 계열도 그런 식으로 동작합니다.)

여기에는 float와 double을 IEEE-754 형식으로 인코딩하는 코드37가 있습니다. (정확히 말하면 대부분의 경우입니다. NaN과 Infinity는 인코딩하지 않지만, 원한다면 수정해서 지원할 수 있습니다.)

#define pack754_32(f) (pack754((f), 32, 8))
#define pack754_64(f) (pack754((f), 64, 11))
#define unpack754_32(i) (unpack754((i), 32, 8))
#define unpack754_64(i) (unpack754((i), 64, 11))

uint64_t pack754(long double f, unsigned bits, unsigned expbits)
{
    long double fnorm;
    int shift;
    long long sign, exp, significand;

    // -1 for sign bit
    unsigned significandbits = bits - expbits - 1;

    if (f == 0.0) return 0; // get this special case out of the way

    // check sign and begin normalization
    if (f < 0) { sign = 1; fnorm = -f; }
    else { sign = 0; fnorm = f; }

    // get the normalized form of f and track the exponent
    shift = 0;
    while(fnorm >= 2.0) { fnorm /= 2.0; shift++; }
    while(fnorm < 1.0) { fnorm *= 2.0; shift--; }
    fnorm = fnorm - 1.0;

    // calculate the binary form (non-float) of the significand data
    significand = fnorm * ((1LL<<significandbits) + 0.5f);

    // get the biased exponent
    exp = shift + ((1<<(expbits-1)) - 1); // shift + bias

    // return the final answer
    return (sign<<(bits-1)) | (exp<<(bits-expbits-1)) | significand;
}

long double unpack754(uint64_t i, unsigned bits, unsigned expbits)
{
    long double result;
    long long shift;
    unsigned bias;

    // -1 for sign bit
    unsigned significandbits = bits - expbits - 1;

    if (i == 0) return 0.0;

    // pull the significand
    result = (i&((1LL<<significandbits)-1)); // mask
    result /= (1LL<<significandbits); // convert back to float
    result += 1.0f; // add the one back on

    // deal with the exponent
    bias = (1<<(expbits-1)) - 1;
    shift = ((i>>significandbits)&((1LL<<expbits)-1)) - bias;
    while(shift > 0) { result *= 2.0; shift--; }
    while(shift < 0) { result /= 2.0; shift++; }

    // sign it
    result *= (i>>(bits-1))&1? -1.0: 1.0;

    return result;
}

I put some handy macros up there at the top for packing and unpacking 32-bit (probably a float) and 64-bit (probably a double) numbers, but the pack754() function could be called directly and told to encode bits-worth of data (expbits of which are reserved for the normalized number’s exponent).

Here’s sample usage:


#include <stdio.h>
#include <stdint.h> // defines uintN_t types
#include <inttypes.h> // defines PRIx macros

int main(void)
{
    float f = 3.1415926, f2;
    double d = 3.14159265358979323, d2;
    uint32_t fi;
    uint64_t di;

    fi = pack754_32(f);
    f2 = unpack754_32(fi);

    di = pack754_64(d);
    d2 = unpack754_64(di);

    printf("float before : %.7f\n", f);
    printf("float encoded: 0x%08" PRIx32 "\n", fi);
    printf("float after  : %.7f\n\n", f2);

    printf("double before : %.20lf\n", d);
    printf("double encoded: 0x%016" PRIx64 "\n", di);
    printf("double after  : %.20lf\n", d2);

    return 0;
}

Unfortunately for you, the compiler is free to put padding all over the place in a struct, and that means you can’t portably send the whole thing over the wire in one chunk. That’s a lot of work, is what you’re thinking. Yes, it is. One thing you can do is write a helper function to help pack the data for you. It’ll be fun! Really!

But if you want to write your own packing utility in C, K&P’s trick is to use variable argument lists to make printf()-like functions to build the packets. Here’s a version I cooked up40 on my own based on that which hopefully will be enough to give you an idea of how such a thing can work.

(This code references the pack754() functions, above. The packi*() functions operate like the familiar htons() family, except they pack into a char array instead of another integer.)

#include <stdio.h>
#include <ctype.h>
#include <stdarg.h>
#include <string.h>

/*
** packi16() -- store a 16-bit int into a char buffer (like htons())
*/
void packi16(unsigned char *buf, unsigned int i)
{
    *buf++ = i>>8; *buf++ = i;
}

/*
** packi32() -- store a 32-bit int into a char buffer (like htonl())
*/
void packi32(unsigned char *buf, unsigned long int i)
{
    *buf++ = i>>24; *buf++ = i>>16;
    *buf++ = i>>8;  *buf++ = i;
}

/*
** packi64() -- store a 64-bit int into a char buffer (like htonl())
*/
void packi64(unsigned char *buf, unsigned long long int i)
{
    *buf++ = i>>56; *buf++ = i>>48;
    *buf++ = i>>40; *buf++ = i>>32;
    *buf++ = i>>24; *buf++ = i>>16;
    *buf++ = i>>8;  *buf++ = i;
}

/*
** unpacki16() -- unpack a 16-bit int from a char buffer (like
**                ntohs())
*/
int unpacki16(unsigned char *buf)
{
    unsigned int i2 = ((unsigned int)buf[0]<<8) | buf[1];
    int i;

    // change unsigned numbers to signed
    if (i2 <= 0x7fffu) { i = i2; }
    else { i = -1 - (unsigned int)(0xffffu - i2); }

    return i;
}

/*
** unpacku16() -- unpack a 16-bit unsigned from a char buffer (like
**                ntohs())
*/
unsigned int unpacku16(unsigned char *buf)
{
    return ((unsigned int)buf[0]<<8) | buf[1];
}

/*
** unpacki32() -- unpack a 32-bit int from a char buffer (like
**                ntohl())
*/
long int unpacki32(unsigned char *buf)
{
    unsigned long int i2 = ((unsigned long int)buf[0]<<24) |
                           ((unsigned long int)buf[1]<<16) |
                           ((unsigned long int)buf[2]<<8)  |
                           buf[3];
    long int i;

    // change unsigned numbers to signed
    if (i2 <= 0x7fffffffu) { i = i2; }
    else { i = -1 - (long int)(0xffffffffu - i2); }

    return i;
}

/*
** unpacku32() -- unpack a 32-bit unsigned from a char buffer (like
**                ntohl())
*/
unsigned long int unpacku32(unsigned char *buf)
{
    return ((unsigned long int)buf[0]<<24) |
           ((unsigned long int)buf[1]<<16) |
           ((unsigned long int)buf[2]<<8)  |
           buf[3];
}

/*
** unpacki64() -- unpack a 64-bit int from a char buffer (like
**                ntohl())
*/
long long int unpacki64(unsigned char *buf)
{
    unsigned long long int i2 =
        ((unsigned long long int)buf[0]<<56) |
        ((unsigned long long int)buf[1]<<48) |
        ((unsigned long long int)buf[2]<<40) |
        ((unsigned long long int)buf[3]<<32) |
        ((unsigned long long int)buf[4]<<24) |
        ((unsigned long long int)buf[5]<<16) |
        ((unsigned long long int)buf[6]<<8)  |
        buf[7];
    long long int i;

    // change unsigned numbers to signed
    if (i2 <= 0x7fffffffffffffffu) { i = i2; }
    else { i = -1 -(long long int)(0xffffffffffffffffu - i2); }

    return i;
}

/*
** unpacku64() -- unpack a 64-bit unsigned from a char buffer (like
**                ntohl())
*/
unsigned long long int unpacku64(unsigned char *buf)
{
    return ((unsigned long long int)buf[0]<<56) |
           ((unsigned long long int)buf[1]<<48) |
           ((unsigned long long int)buf[2]<<40) |
           ((unsigned long long int)buf[3]<<32) |
           ((unsigned long long int)buf[4]<<24) |
           ((unsigned long long int)buf[5]<<16) |
           ((unsigned long long int)buf[6]<<8)  |
           buf[7];
}

/*
** pack() -- store data dictated by the format string in the buffer
**
**   bits |signed   unsigned   float   string
**   -----+----------------------------------
**      8 |   c        C
**     16 |   h        H         f
**     32 |   l        L         d
**     64 |   q        Q         g
**      - |                               s
**
**  (16-bit unsigned length is automatically prepended to strings)
*/

unsigned int pack(unsigned char *buf, char *format, ...)
{
    va_list ap;

    signed char c;              // 8-bit
    unsigned char C;

    int h;                      // 16-bit
    unsigned int H;

    long int l;                 // 32-bit
    unsigned long int L;

    long long int q;            // 64-bit
    unsigned long long int Q;

    float f;                    // floats
    double d;
    long double g;
    unsigned long long int fhold;

    char *s;                    // strings
    unsigned int len;

    unsigned int size = 0;

    va_start(ap, format);

    for(; *format != '\0'; format++) {
        switch(*format) {
        case 'c': // 8-bit
            size += 1;
            c = (signed char)va_arg(ap, int); // promoted
            *buf++ = c;
            break;

        case 'C': // 8-bit unsigned
            size += 1;
            C = (unsigned char)va_arg(ap, unsigned int); // promoted
            *buf++ = C;
            break;

        case 'h': // 16-bit
            size += 2;
            h = va_arg(ap, int);
            packi16(buf, h);
            buf += 2;
            break;

        case 'H': // 16-bit unsigned
            size += 2;
            H = va_arg(ap, unsigned int);
            packi16(buf, H);
            buf += 2;
            break;

        case 'l': // 32-bit
            size += 4;
            l = va_arg(ap, long int);
            packi32(buf, l);
            buf += 4;
            break;

        case 'L': // 32-bit unsigned
            size += 4;
            L = va_arg(ap, unsigned long int);
            packi32(buf, L);
            buf += 4;
            break;

        case 'q': // 64-bit
            size += 8;
            q = va_arg(ap, long long int);
            packi64(buf, q);
            buf += 8;
            break;

        case 'Q': // 64-bit unsigned
            size += 8;
            Q = va_arg(ap, unsigned long long int);
            packi64(buf, Q);
            buf += 8;
            break;

        case 'f': // float-16
            size += 2;
            f = (float)va_arg(ap, double); // promoted
            fhold = pack754_16(f); // convert to IEEE 754
            packi16(buf, fhold);
            buf += 2;
            break;

        case 'd': // float-32
            size += 4;
            d = va_arg(ap, double);
            fhold = pack754_32(d); // convert to IEEE 754
            packi32(buf, fhold);
            buf += 4;
            break;

        case 'g': // float-64
            size += 8;
            g = va_arg(ap, long double);
            fhold = pack754_64(g); // convert to IEEE 754
            packi64(buf, fhold);
            buf += 8;
            break;

        case 's': // string
            s = va_arg(ap, char*);
            len = strlen(s);
            size += len + 2;
            packi16(buf, len);
            buf += 2;
            memcpy(buf, s, len);
            buf += len;
            break;
        }
    }

    va_end(ap);

    return size;
}

/*
** unpack() -- unpack data dictated by the format string into the
**             buffer
**
**   bits |signed   unsigned   float   string
**   -----+----------------------------------
**      8 |   c        C
**     16 |   h        H         f
**     32 |   l        L         d
**     64 |   q        Q         g
**      - |                               s
**
**  (string is extracted based on its stored length, but 's' can be
**  prepended with a max length)
*/
void unpack(unsigned char *buf, char *format, ...)
{
    va_list ap;

    signed char *c;              // 8-bit
    unsigned char *C;

    int *h;                      // 16-bit
    unsigned int *H;

    long int *l;                 // 32-bit
    unsigned long int *L;

    long long int *q;            // 64-bit
    unsigned long long int *Q;

    float *f;                    // floats
    double *d;
    long double *g;
    unsigned long long int fhold;

    char *s;
    unsigned int len, maxstrlen=0, count;

    va_start(ap, format);

    for(; *format != '\0'; format++) {
        switch(*format) {
        case 'c': // 8-bit
            c = va_arg(ap, signed char*);
            if (*buf <= 0x7f) { *c = *buf;} // re-sign
            else { *c = -1 - (unsigned char)(0xffu - *buf); }
            buf++;
            break;

        case 'C': // 8-bit unsigned
            C = va_arg(ap, unsigned char*);
            *C = *buf++;
            break;

        case 'h': // 16-bit
            h = va_arg(ap, int*);
            *h = unpacki16(buf);
            buf += 2;
            break;

        case 'H': // 16-bit unsigned
            H = va_arg(ap, unsigned int*);
            *H = unpacku16(buf);
            buf += 2;
            break;

        case 'l': // 32-bit
            l = va_arg(ap, long int*);
            *l = unpacki32(buf);
            buf += 4;
            break;

        case 'L': // 32-bit unsigned
            L = va_arg(ap, unsigned long int*);
            *L = unpacku32(buf);
            buf += 4;
            break;

        case 'q': // 64-bit
            q = va_arg(ap, long long int*);
            *q = unpacki64(buf);
            buf += 8;
            break;

        case 'Q': // 64-bit unsigned
            Q = va_arg(ap, unsigned long long int*);
            *Q = unpacku64(buf);
            buf += 8;
            break;

        case 'f': // float
            f = va_arg(ap, float*);
            fhold = unpacku16(buf);
            *f = unpack754_16(fhold);
            buf += 2;
            break;

        case 'd': // float-32
            d = va_arg(ap, double*);
            fhold = unpacku32(buf);
            *d = unpack754_32(fhold);
            buf += 4;
            break;

        case 'g': // float-64
            g = va_arg(ap, long double*);
            fhold = unpacku64(buf);
            *g = unpack754_64(fhold);
            buf += 8;
            break;

        case 's': // string
            s = va_arg(ap, char*);
            len = unpacku16(buf);
            buf += 2;
            if (maxstrlen > 0 && len > maxstrlen)
                count = maxstrlen - 1;
            else
                count = len;
            memcpy(s, buf, count);
            s[count] = '\0';
            buf += len;
            break;

        default:
            if (isdigit(*format)) { // track max str len
                maxstrlen = maxstrlen * 10 + (*format-'0');
            }
        }

        if (!isdigit(*format)) maxstrlen = 0;
    }

    va_end(ap);
}

그리고 여기에는 데모 프로그램41이 있는데, 위 코드를 사용해 데이터를 buf에 패킹한 뒤 다시 변수들로 언패킹합니다. 문자열 인수(형식 지정자 “s”)와 함께 unpack()를 호출할 때는 버퍼 오버런을 막기 위해 앞에 최대 길이를 적어 두는 것이 좋습니다. 예를 들어 “96s”처럼요. 네트워크에서 받은 데이터를 언패킹할 때는 특히 조심해야 합니다. 악의적인 사용자가 시스템을 공격하려고 잘못 구성된 패킷을 보낼 수도 있으니까요.

#include <stdio.h>
#include <stdint.h>
#include <inttypes.h>

// If you have a C23 compiler
#if __STDC_VERSION__ >= 202311L
#include <stdfloat.h>
#else
// Otherwise let's define our own.
// Varies for different architectures! But you're probably:
typedef float float32_t;
typedef double float64_t;
#endif

int main(void)
{
    uint8_t buf[1024];
    int8_t magic;
    int16_t monkeycount;
    int32_t altitude;
    float32_t absurdityfactor;
    char *s = "Great unmitigated Zot!  You've found the Runestaff!";
    char s2[96];
    int16_t packetsize, ps2;

    packetsize = pack(buf, "chhlsf", (int8_t)'B', (int16_t)0,
            (int16_t)37, (int32_t)-5, s, (float32_t)-3490.6677);
    packi16(buf+1, packetsize); // store packet size for kicks

    printf("packet is %" PRId32 " bytes\n", packetsize);

    unpack(buf, "chhl96sf", &magic, &ps2, &monkeycount, &altitude,
            s2, &absurdityfactor);

    printf("'%c' %" PRId32" %" PRId16 " %" PRId32
            " \"%s\" %f\n", magic, ps2, monkeycount,
            altitude, s2, absurdityfactor);
}

직접 코드를 만들든 남이 만든 코드를 쓰든, 매번 비트를 손으로 하나하나 패킹하는 것보다는 버그를 줄이기 위해 범용 데이터 패킹 루틴 집합을 마련해 두는 것이 좋습니다.

그렇다면 데이터를 패킹할 때 좋은 형식은 무엇일까요? 아주 좋은 질문입니다. 다행히도 External Data Representation Standard인 RFC 450642이 이미 부동소수점형, 정수형, 배열, 원시 데이터 등 다양한 타입에 대한 바이너리 형식을 정의하고 있습니다. 직접 데이터 형식을 설계할 생각이라면 여기에 맞추는 것을 권합니다. 물론 꼭 그래야만 하는 것은 아닙니다. 패킷 경찰이 바로 문밖에서 기다리고 있는 건 아니니까요. 적어도 제 생각에는 그렇습니다.

어쨌든 데이터를 보내기 전에 어떤 방식으로든 인코딩하는 것이 올바른 방법입니다!

7.6 데이터 캡슐화의 후속편

도대체 데이터를 캡슐화한다는 게 실제로 무슨 뜻일까요? 가장 단순한 경우에는 식별 정보나 패킷 길이, 혹은 둘 다를 담은 헤더를 앞에 붙이는 것을 의미합니다.

그렇다면 헤더는 어떤 모양이어야 할까요? 결국 프로젝트를 완성하는 데 필요하다고 생각하는 정보를 표현하는 바이너리 데이터일 뿐입니다.

와, 정말 모호하죠.

좋습니다. 예를 들어 SOCK_STREAM을 사용하는 다중 사용자 채팅 프로그램이 있다고 해 봅시다. 사용자가 무언가를 입력해 말하면, 서버로는 두 가지 정보가 전송되어야 합니다. 무엇을 말했는지, 그리고 누가 말했는지입니다.

여기까지는 괜찮죠? 이제 “문제가 뭔데요?”라고 묻고 있을 겁니다.

문제는 메시지 길이가 제각각이라는 점입니다. “tom”이라는 사용자는 “Hi”라고 말할 수 있고, “Benjamin”이라는 사용자는 “Hey guys what is up?”이라고 말할 수도 있습니다.

그래서 이런 데이터가 들어오는 대로 클라이언트들에게 send()된다고 해 봅시다. 그러면 송신 데이터 스트림은 이런 모습이 됩니다.

t o m H i B e n j a m i n H e y g u y s w h a t i s u p ?

이런 식으로 이어집니다. 그렇다면 클라이언트는 어디서 한 메시지가 시작하고 다음 메시지가 끝나는지 어떻게 알까요? 원한다면 모든 메시지를 같은 길이로 맞춰 놓고 위에서 구현한 sendall()만 계속 호출할 수도 있습니다. 하지만 그건 대역폭 낭비입니다! “tom”이 “Hi”라고 말하는데 1024바이트를 send()하고 싶지는 않잖아요.

그래서 작은 헤더와 패킷 구조로 데이터를 캡슐화합니다. 클라이언트와 서버는 모두 이 데이터를 어떻게 패킹하고 언패킹(“marshal”과 “unmarshal”이라고도 부릅니다)하는지 알고 있습니다. 눈치채셨겠지만, 우리는 지금 클라이언트와 서버가 어떻게 통신하는지 설명하는 하나의 프로토콜을 정의하기 시작한 것입니다!

이 경우 사용자 이름은 '\0'으로 패딩된 고정 길이 8문자라고 가정해 봅시다. 그리고 실제 데이터는 최대 128문자까지 가능한 가변 길이라고 하겠습니다. 이런 상황에서 사용할 수 있는 예시 패킷 구조를 봅시다.

  1. len (1바이트, unsigned): 8바이트 사용자 이름과 채팅 데이터를 포함한 패킷 전체 길이

  2. name (8바이트): 필요하면 NUL로 패딩된 사용자 이름

  3. chatdata (n바이트): 실제 데이터. 최대 128바이트까지이며, 패킷 길이는 이 데이터 길이에 위의 이름 필드 길이 8을 더해 계산합니다.

왜 8바이트와 128바이트를 골랐냐고요? 그냥 적당히 충분할 것 같아서 정한 것입니다. 하지만 여러분 상황에서는 8바이트가 너무 짧을 수도 있고, 30바이트짜리 이름 필드가 필요할 수도 있습니다. 선택은 여러분 몫입니다.

위 패킷 정의를 따르면 첫 번째 패킷은 다음과 같은 정보로 구성됩니다. (16진수와 ASCII 표기)

   0A     74 6F 6D 00 00 00 00 00      48 69
(length)  T  o  m    (padding)         H  i

두 번째도 비슷합니다.

   18     42 65 6E 6A 61 6D 69 6E      48 65 79 20 67 75 79 73 20 77 ...
(length)  B  e  n  j  a  m  i  n       H  e  y     g  u  y  s     w  ...

(물론 길이는 네트워크 바이트 순서로 저장됩니다. 이 경우엔 1바이트뿐이라 큰 차이가 없지만, 일반적으로는 패킷 안의 모든 바이너리 정수 값을 네트워크 바이트 순서로 저장하는 것이 좋습니다.)

이 데이터를 보낼 때는 여러 번의 send() 호출이 필요하더라도 데이터가 모두 전송되었음을 보장할 수 있도록 위의 sendall()과 비슷한 함수를 사용하는 것이 안전합니다.

마찬가지로 이 데이터를 받을 때도 약간의 추가 작업이 필요합니다. 안전하게 가려면 패킷 일부만 받을 수도 있다고 가정해야 합니다. 예를 들어 위의 Benjamin 패킷에서 “18 42 65 6E 6A”까지만 이번 recv() 호출에서 들어올 수도 있습니다. 그러면 패킷이 완전히 도착할 때까지 recv()를 반복해서 호출해야 합니다.

어떻게 하냐고요? 패킷 맨 앞에 전체 길이가 붙어 있으므로, 패킷이 완성되려면 총 몇 바이트를 받아야 하는지 우리는 알고 있습니다. 또한 최대 패킷 크기도 1+8+128, 즉 137바이트라는 것을 알고 있죠. (우리가 그렇게 정의했으니까요.)

여기서 할 수 있는 방법은 몇 가지가 있습니다. 모든 패킷이 길이 값으로 시작한다는 것을 아니까, 먼저 패킷 길이만 얻기 위해 recv()를 한 번 호출할 수 있습니다. 그런 다음 그 길이를 알게 되면 나머지 길이만큼 정확히 지정해서 다시 호출하고, 필요하면 여러 번 반복해서 완전한 패킷을 받으면 됩니다. 이 방법의 장점은 패킷 하나를 담을 버퍼만 있으면 된다는 것이고, 단점은 데이터를 모두 얻기 위해 최소 두 번은 recv()를 호출해야 한다는 것입니다.

다른 방법은 그냥 recv()를 호출하면서 한 번에 받을 최대 바이트 수를 패킷 최대 크기로 두는 것입니다. 그리고 받아 온 데이터를 작업 버퍼 뒤에 계속 붙인 다음, 패킷이 완성되었는지 검사하면 됩니다. 물론 다음 패킷의 일부까지 함께 들어올 수 있으니 그것을 담을 공간도 필요합니다.

그래서 패킷 두 개를 담을 수 있을 만큼 큰 배열을 선언할 수 있습니다. 이 배열이 패킷을 조립하는 작업 버퍼가 됩니다.

데이터를 recv()할 때마다 작업 버퍼 뒤에 덧붙이고, 패킷이 완성되었는지 확인합니다. 즉, 버퍼 안 바이트 수가 헤더에 기록된 길이 이상인지 보면 됩니다. (길이 바이트 자체는 헤더 길이에 포함되지 않으므로 +1을 해야 합니다.) 버퍼 안 바이트 수가 1보다 작다면 당연히 패킷은 완성되지 않은 것입니다. 다만 첫 바이트가 아직 쓰레기 값일 수 있으므로, 올바른 패킷 길이를 판단할 수 없다는 점에서 이 경우는 특별히 처리해야 합니다.

패킷이 완성되면 원하는 작업을 하면 됩니다. 사용하고, 작업 버퍼에서는 제거하면 됩니다.

휴, 머릿속으로 잘 따라오고 있나요? 그런데 여기서 끝이 아닙니다. 한 번의 recv() 호출에서 첫 패킷 끝을 넘어 다음 패킷 일부까지 읽어 올 수도 있습니다. 즉 작업 버퍼에는 완전한 패킷 하나와 다음 패킷의 불완전한 조각이 같이 들어 있을 수 있다는 뜻입니다. 이런 상황을 대비하려고 작업 버퍼를 두 개 패킷이 들어갈 정도로 크게 잡아 둔 것이죠.

첫 번째 패킷 길이는 헤더를 보고 알고 있고, 작업 버퍼 전체 바이트 수도 추적하고 있으므로, 버퍼 안 데이터 중 두 번째 (불완전한) 패킷에 해당하는 바이트 수를 계산할 수 있습니다. 첫 번째 패킷 처리가 끝나면 작업 버퍼에서 그것을 지우고, 두 번째 패킷의 남은 조각을 버퍼 맨 앞으로 당겨 놓으면 다음 recv()를 받을 준비가 됩니다.

(몇몇 독자분들은 부분적으로 받은 두 번째 패킷을 버퍼 맨 앞으로 실제로 옮기는 데 시간이 걸린다는 점을 눈치챘을 겁니다. 원형 버퍼를 쓰면 이런 복사가 필요 없도록 코드를 짤 수도 있습니다. 하지만 안타깝게도 원형 버퍼 이야기는 이 글 범위를 벗어납니다. 더 궁금하다면 자료구조 책을 들고 이어서 공부해 보세요.)

제가 쉽다고 안 했던가요? 아니, 하긴 쉽다고 하긴 했죠. 그리고 실제로 그렇습니다. 연습만 좀 하면 곧 자연스럽게 감이 올 겁니다. 엑스칼리버를 걸고 맹세합니다!

7.7 브로드캐스트 패킷—Hello, World!

지금까지 이 가이드는 한 호스트에서 다른 한 호스트로 데이터를 보내는 이야기만 해 왔습니다. 하지만 적절한 권한만 있다면, 정말로, 동시에 여러 호스트로 데이터를 보낼 수도 있습니다!

UDP(UDP만, TCP는 아닙니다)와 표준 IPv4에서는 이것을 브로드캐스팅이라는 메커니즘으로 수행합니다. IPv6에서는 브로드캐스팅이 지원되지 않아서 대신 더 우수한 경우가 많은 멀티캐스팅 기법을 사용해야 합니다. 다만 안타깝게도 여기서는 다루지 않겠습니다. 미래 이야기는 이쯤 하고, 우리는 아직 32비트 현재에 살고 있으니까요.

하지만 잠깐! 그냥 막 브로드캐스트를 날릴 수는 없습니다. 네트워크에 브로드캐스트 패킷을 보내기 전에 반드시 소켓 옵션 SO_BROADCAST를 설정해야 합니다. 마치 미사일 발사 스위치 위의 플라스틱 보호 덮개 같은 거죠. 그만큼 큰 힘을 여러분 손에 쥐고 있다는 뜻입니다!

농담은 제쳐 두고, 브로드캐스트 패킷에는 실제 위험도 있습니다. 브로드캐스트를 받은 모든 시스템은 데이터가 어느 포트로 가야 하는지 알아내기 위해 캡슐화의 여러 층을 전부 벗겨 내야 합니다. 그런 다음 데이터를 넘겨주거나 폐기하죠. 어느 쪽이든 각 머신마다 일이 많습니다. 그리고 그 대상이 로컬 네트워크의 모든 머신이니, 엄청나게 많은 불필요한 작업이 생길 수 있습니다. 게임 Doom이 처음 나왔을 때도 네트워크 코드에 대한 불만 중 하나가 바로 이것이었습니다.

고양이 가죽을 벗기는 방법이 하나만 있는 건 아니라는 말이 있죠. 잠깐, 정말 그런 말이 맞나? 아무튼 브로드캐스트 패킷을 보내는 방법도 하나만 있는 것은 아닙니다. 핵심으로 바로 들어가면, 브로드캐스트 메시지의 목적지 주소를 어떻게 지정할까요? 흔히 쓰는 방법은 두 가지입니다.

  1. 특정 서브넷의 브로드캐스트 주소로 데이터를 보냅니다. 이것은 해당 서브넷의 네트워크 번호에서 호스트 부분 비트를 모두 1로 만든 주소입니다. 예를 들어 제 집 네트워크가 192.168.1.0이고 넷마스크가 255.255.255.0이라면, 주소 마지막 바이트가 호스트 번호가 됩니다. (넷마스크 기준으로 앞의 세 바이트는 네트워크 번호이기 때문입니다.) 따라서 브로드캐스트 주소는 192.168.1.255가 됩니다. Unix에서는 ifconfig 명령으로 이런 정보를 확인할 수 있습니다. (궁금하다면 브로드캐스트 주소를 구하는 비트 연산은 network_number OR (NOT netmask)입니다.) 이런 브로드캐스트 패킷은 로컬 네트워크뿐 아니라 원격 네트워크로도 보낼 수 있지만, 목적지 라우터가 패킷을 버릴 위험이 있습니다. (라우터가 안 버린다면 누군가가 LAN 전체를 브로드캐스트 트래픽으로 도배할 수도 있으니까요.)

  2. “전역” 브로드캐스트 주소로 보냅니다. 이것은 255.255.255.255, 즉 INADDR_BROADCAST입니다. 많은 머신은 이 주소를 자동으로 네트워크 번호와 비트 AND 해서 네트워크 브로드캐스트 주소로 바꾸지만, 그렇지 않은 시스템도 있습니다. 시스템마다 다릅니다. 역설적이게도 라우터는 이런 종류의 브로드캐스트 패킷을 로컬 네트워크 밖으로는 전달하지 않습니다.

그렇다면 SO_BROADCAST 소켓 옵션을 설정하지 않은 채 브로드캐스트 주소로 데이터를 보내면 어떻게 될까요? 익숙한 talker와 listener를 다시 켜서 확인해 봅시다.

$ talker 192.168.1.2 foo
sent 3 bytes to 192.168.1.2
$ talker 192.168.1.255 foo
sendto: Permission denied
$ talker 255.255.255.255 foo
sendto: Permission denied

보시다시피 전혀 행복하지 않습니다. 이유는 SO_BROADCAST 소켓 옵션을 설정하지 않았기 때문입니다. 그것만 설정하면 이제 원하는 곳으로 sendto()할 수 있습니다!

사실 브로드캐스트 가능한 UDP 애플리케이션과 그렇지 않은 애플리케이션의 유일한 차이가 바로 이것입니다. 그러니 예전의 talker 프로그램에 SO_BROADCAST를 설정하는 코드만 추가해 봅시다. 이 프로그램 이름은 broadcaster.c43라고 하겠습니다.

/*
** broadcaster.c -- a datagram "client" like talker.c, except
**                  this one can broadcast
*/

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <errno.h>
#include <string.h>
#include <sys/types.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>
#include <netdb.h>

#define SERVERPORT 4950    // the port users will be connecting to

int main(int argc, char *argv[])
{
    int sockfd;
    struct sockaddr_in their_addr; // connector's address info
    struct hostent *he;
    int numbytes;
    int broadcast = 1;
    //char broadcast = '1'; // if that doesn't work, try this

    if (argc != 3) {
        fprintf(stderr,"usage: broadcaster hostname message\n");
        exit(1);
    }

    if ((he=gethostbyname(argv[1])) == NULL) {  // get the host info
        perror("gethostbyname");
        exit(1);
    }

    if ((sockfd = socket(PF_INET, SOCK_DGRAM, 0)) == -1) {
        perror("socket");
        exit(1);
    }

    // this call is what allows broadcast packets to be sent:
    if (setsockopt(sockfd, SOL_SOCKET, SO_BROADCAST, &broadcast,
        sizeof broadcast) == -1) {
        perror("setsockopt (SO_BROADCAST)");
        exit(1);
    }

    their_addr.sin_family = AF_INET;     // host byte order
    their_addr.sin_port = htons(SERVERPORT); // network byte order
    their_addr.sin_addr = *((struct in_addr *)he->h_addr);
    memset(their_addr.sin_zero, '\0', sizeof their_addr.sin_zero);

    numbytes = sendto(sockfd, argv[2], strlen(argv[2]), 0,
             (struct sockaddr *)&their_addr, sizeof their_addr);

    if (numbytes == -1) {
        perror("sendto");
        exit(1);
    }

    printf("sent %d bytes to %s\n", numbytes,
        inet_ntoa(their_addr.sin_addr));

    close(sockfd);

    return 0;
}

이것과 “보통의” UDP 클라이언트/서버 상황 사이에 무엇이 다를까요? 아무것도 없습니다! (이번 경우에는 클라이언트가 브로드캐스트 패킷을 보낼 수 있다는 점만 제외하면요.) 그러니 한 창에서는 예전 UDP listener를 실행하고, 다른 창에서는 broadcaster를 실행해 보세요. 이제 위에서 실패했던 전송들이 모두 가능해질 것입니다.

$ broadcaster 192.168.1.2 foo
sent 3 bytes to 192.168.1.2
$ broadcaster 192.168.1.255 foo
sent 3 bytes to 192.168.1.255
$ broadcaster 255.255.255.255 foo
sent 3 bytes to 255.255.255.255

그러면 listener가 패킷을 받았다고 응답하는 모습도 보여야 합니다. (listener가 반응하지 않는다면 IPv6 주소에 바인드되어 있기 때문일 수 있습니다. listener.c 안의 AF_INET6를 AF_INET으로 바꿔 IPv4를 강제해 보세요.)

꽤 신기하죠. 그런데 이번에는 같은 네트워크의 옆 머신에서도 listener를 하나 더 띄워서 두 머신에서 각각 하나씩 실행해 보세요. 그리고 브로드캐스트 주소로 broadcaster를 다시 실행해 보세요. 어라! sendto()는 한 번만 호출했는데 두 listener가 모두 패킷을 받습니다. 멋지죠.

listener가 직접 보낸 데이터는 받는데 브로드캐스트 주소로 보낸 데이터는 받지 못한다면, 로컬 머신 방화벽이 패킷을 막고 있을 가능성이 있습니다. (네, Pat과 Bapper, 제 예제 코드가 왜 안 됐는지 저보다 먼저 알아차려 줘서 고맙습니다. 가이드에 이름을 넣어 주겠다고 했었죠. 그래서 여기 적어 둡니다.)

다시 한 번 말하지만 브로드캐스트 패킷은 조심해서 사용해야 합니다. LAN의 모든 머신은 그 패킷을 실제로 recvfrom()하든 말든 일단 처리해야 하기 때문에, 전체 네트워크에 상당한 부하를 줄 수 있습니다. 반드시 필요한 경우에만, 적절하게 사용해야 합니다.
